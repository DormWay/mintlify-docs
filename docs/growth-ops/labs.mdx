# Labs Admin Dashboard

The **Labs Admin Dashboard** is where you manage the DormWay Labs program. This includes reviewing applications, creating tasks for the pool, approving submissions, and tracking member performance.

<Info>
  **New in 2026:** Labs now uses a **self-service task pool model**. Students apply to join, browse available tasks, and claim them first-come-first-served. This replaces the old manual assignment model.
</Info>

---

## Accessing the Dashboard

<Steps>
  <Step title="Go to Admin Dashboard">
    Navigate to `https://admin.dormway.app` and sign in
  </Step>
  <Step title="Open Growth Section">
    Click **Growth** in the sidebar, then **Labs**
  </Step>
  <Step title="Choose Your Tab">
    - **Applications** - Review pending applications
    - **Members** - View all Labs members
    - **Task Pool** - Manage available tasks
    - **Submissions** - Review completed work
    - **Analytics** - Track program metrics
  </Step>
</Steps>

---

## Daily Workflows

### 1. Review Applications

New applications appear in the **Applications** tab. Review them daily to keep the pipeline moving.

<Steps>
  <Step title="Open Pending Applications">
    Filter by **Status: Pending** to see applications awaiting review
  </Step>
  <Step title="Review Application Details">
    Check:
    - **Motivation**: Why do they want to join?
    - **Social Reach**: How connected are they on campus?
    - **Campus**: Are they from a campus we're targeting?
  </Step>
  <Step title="Approve or Reject">
    - **Approve**: Click **Approve** to add them as an active member
    - **Reject**: Click **Reject** and provide a reason (they can reapply later)
  </Step>
</Steps>

**Approval criteria:**
- Clear motivation to help improve DormWay
- From an active campus (or campus we want to expand to)
- No red flags in application

<Tip>
  **Be generous with approvals** - the task pool model is self-regulating. Poor performers naturally do fewer tasks and earn less.
</Tip>

---

### 2. Create Task Pool Items

The **Task Pool** is where you add tasks that members can claim. Keep the pool stocked with a variety of tasks.

<Steps>
  <Step title="Click Add Task">
    In the Task Pool tab, click **Add Task**
  </Step>
  <Step title="Fill Task Details">
    - **Title**: Clear, descriptive name
    - **Description**: Detailed instructions
    - **Type**: Syllabus Upload, Bug Report, QA Testing, Research, Other
    - **Payout (cents)**: How much this task pays (e.g., 500 = $5.00)
    - **Quantity**: How many of this task are available (e.g., 10 syllabus uploads)
    - **Deadline**: When tasks must be submitted by
    - **Campus** (optional): Restrict to specific campus
  </Step>
  <Step title="Save to Pool">
    Click **Create** - the task immediately appears in members' task pools
  </Step>
</Steps>

**Example tasks:**

| Type | Title | Payout | Quantity | Description |
|------|-------|--------|----------|-------------|
| Syllabus Upload | Upload STEM syllabi | $3 | 20 | Upload any STEM course syllabus from your campus |
| Bug Report | Find iOS bugs | $5-20 | Unlimited | Report bugs in the iOS app. Payout based on severity. |
| QA Testing | Test schedule sync | $10 | 5 | Test the Canvas schedule sync flow and report issues |

<Warning>
  **Real-time updates:** When you add tasks, members see them immediately. Make sure details are correct before saving.
</Warning>

---

### 3. Review Submissions

Members submit completed work through the dashboard. Review submissions in the **Submissions** tab.

<Steps>
  <Step title="Open Pending Submissions">
    Filter by **Status: Submitted** to see work awaiting review
  </Step>
  <Step title="Review the Deliverable">
    Check the submitted URL or notes:
    - **Syllabus uploads**: Is the PDF readable? Correct course?
    - **Bug reports**: Is it reproducible? Well-documented?
    - **Other tasks**: Does it meet the requirements?
  </Step>
  <Step title="Approve or Reject">
    - **Approve**: Mark as approved. Task is queued for payment.
    - **Reject**: Provide feedback. Member can fix and resubmit.
  </Step>
</Steps>

**Quality checklist for syllabi:**
- [ ] PDF is readable (not blurry scan)
- [ ] Contains grading breakdown
- [ ] Contains course schedule
- [ ] Matches the course it was uploaded for
- [ ] Not a duplicate

## Task Pool Management

### Task Types

| Type | Use Case | Typical Payout |
|------|----------|----------------|
| **Syllabus Upload** | Collect syllabi for courses | $2-5 per syllabus |
| **Bug Report** | Find and document bugs | $5-20 based on severity |
| **QA Testing** | Test specific features | $5-15 per test session |
| **Research** | User interviews, surveys | $10-25 per task |
| **Other** | Custom tasks | Varies |

### Setting Payouts

- **Start low, adjust up** if tasks aren't getting claimed
- **Higher payouts** for harder or more urgent tasks
- **Campus-specific premiums** for high-priority campuses

### Managing Quantity

- **Unlimited tasks**: Set quantity to a high number (999)
- **Limited tasks**: Set exact quantity needed
- **Quantity decreases** as members claim tasks

### Deadlines

- **Short deadlines** (24-48h) for urgent tasks
- **Longer deadlines** (1 week) for complex tasks
- **Rolling deadlines** - tasks return to pool if not submitted in time

## Member Management

### Viewing Members

The **Members** tab shows all Labs members with their stats:
- **Status**: Active, Inactive, Pending
- **Tasks Completed**: All-time count
- **Total Earned**: All-time earnings
- **Campus**: Their home campus
- **Joined**: When they joined Labs

### Member Actions

- **View Profile**: See detailed history
- **Deactivate**: Remove from Labs (they can reapply)
- **Add Note**: Add internal notes about the member

### Performance Tracking

Sort and filter by performance metrics:
- **Top earners**: Members completing the most tasks
- **Inactive**: Members who haven't claimed tasks recently
- **New**: Recently approved members

## Analytics

The **Analytics** tab provides program insights:

### Key Metrics

| Metric | Description |
|--------|-------------|
| **Active Members** | Members who claimed tasks in last 30 days |
| **Tasks Completed (Week)** | Total tasks approved this week |
| **Total Paid (Week)** | Amount paid out this week |
| **Avg. Time to Complete** | How long tasks take to complete |
| **Approval Rate** | % of submissions approved first try |

### Campus Breakdown

See which campuses are most active:
- Syllabi uploaded per campus
- Active members per campus
- Helps identify where to recruit more members

## Real-Time Features

The Labs system includes real-time updates:

### For Admins
- **New submissions** appear immediately in the Submissions tab
- **Task claims** update the pool count in real-time
- **Badge notifications** show pending items needing attention

### For Members
- **New tasks** appear in their pool immediately
- **Claimed tasks** disappear from pool for others
- **Submission status** updates when you approve/reject

## Troubleshooting

<AccordionGroup>
  <Accordion title="Task not appearing in members' pools">
    **Check:**
    - Is the task status "Active"?
    - Is the deadline in the future?
    - Is quantity > 0?
    - If campus-restricted, is the member from that campus?
  </Accordion>

  <Accordion title="Member can't claim tasks">
    **Check:**
    - Is their member status "Active"?
    - Have they reached a claim limit? (if implemented)
    - Are there actually tasks available in the pool?
  </Accordion>

  <Accordion title="Submission not showing up">
    **Check:**
    - Did the member click Submit?
    - Check the task's submissions in the Task Pool detail view
    - Member may have submitted to wrong task
  </Accordion>

  <Accordion title="Payment not processing">
    **Check:**
    - Is the task marked as "Approved"?
    - Has the payout been marked as processed?
    - Check member's payment info is set up
  </Accordion>
</AccordionGroup>
