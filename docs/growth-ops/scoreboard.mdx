## Overview

The **Growth Scoreboard** is DormWay's pre-computed analytics dashboard tracking key growth metrics on a weekly basis. It aggregates data from multiple sources (ambassadors, campus outreach, Labs, organic) to provide a unified view of growth performance.

  </Card>
  <Card title="Day 7 Return" icon="rotate-right">

</CardGroup>

## Database Schema

### `growth_scoreboard` Table

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Primary key |
| `week_start` | DATE | Monday of the week (ISO week start) |
| `source` | TEXT | Traffic source (see below) |
| `campus_id` | UUID | Optional: Campus-level breakdown |
| `reach` | INTEGER | Total reach (impressions, views, etc.) |
| `clicks` | INTEGER | Total clicks on campaign links |
| `signups` | INTEGER | New accounts created |
| `wfau` | INTEGER | Weekly First-time Active Users |
| `day_7_return` | INTEGER | Users who returned on day 7 |
| `day_7_return_pct` | NUMERIC(5,2) | `(day_7_return / wfau) * 100` |
| `platform_split` | JSONB | `{ "ios": 50, "web": 30, "android": 20 }` |
| `metadata` | JSONB | Additional context (campaign IDs, channel breakdowns) |
| `created_at` | TIMESTAMPTZ | Record creation time |
| `updated_at` | TIMESTAMPTZ | Last update time |

<Info>
  **Unique Constraint**: One row per `(week_start, source, campus_id)` combination. Null `campus_id` means aggregate across all campuses.
</Info>

## Key Metrics Explained

### Signups

**Definition**: New accounts created during the week.

**Calculation**:
```sql
SELECT COUNT(*)
FROM accounts
WHERE created_at >= '2026-01-13' -- Monday
  AND created_at < '2026-01-20'   -- Next Monday
  AND referral_code_used LIKE 'ambassador_%'; -- Example: ambassador source
```

**Target**: Varies by growth stage. Early stage: 50-100/week. Scale stage: 500+/week.

---

### WFAU (Weekly First-time Active Users)

**Definition**: Users who completed onboarding AND performed at least one meaningful action (uploaded syllabus, viewed schedule, synced calendar) within their first week.

**Calculation**:
```sql
-- Users who became "active" for the first time this week
SELECT COUNT(DISTINCT a.id)
FROM accounts a
WHERE a.created_at >= '2026-01-13'
  AND EXISTS (
    -- Check for meaningful activity (customize per product definition)
    SELECT 1 FROM service_data sd
    WHERE sd.user_id = a.id
      AND sd.fetched_at <= a.created_at + INTERVAL '7 days'
      AND sd.method IN ('dayplan', 'processed_syllabus_crew', 'calendar_events')
  );
```

**Target**: 70-90% of signups should become WFAU. If < 50%, onboarding is broken.

---

### Day 7 Return Rate

**Definition**: Percentage of WFAU who returned to the app on Day 7 after signup.

**Why Day 7?**: Strong predictor of long-term retention. Users who return on D7 are 5x more likely to be retained at D30.

**Calculation**:
```sql
-- Users who logged in on their 7th day
WITH day_7_cohort AS (
  SELECT a.id, a.created_at::date as signup_date
  FROM accounts a
  WHERE a.created_at >= '2026-01-06' -- 7 days before week_start
    AND a.created_at < '2026-01-13'  -- Week prior to current week
)
SELECT
  COUNT(DISTINCT dc.id) FILTER (WHERE ul.timestamp::date = dc.signup_date + INTERVAL '7 days') as day_7_returns,
  COUNT(DISTINCT dc.id) as total_wfau,
  (COUNT(DISTINCT dc.id) FILTER (WHERE ul.timestamp::date = dc.signup_date + INTERVAL '7 days')::float /
   COUNT(DISTINCT dc.id)) * 100 as day_7_return_pct
FROM day_7_cohort dc
LEFT JOIN user_locations ul ON ul.user_id = dc.id
  AND ul.timestamp::date = dc.signup_date + INTERVAL '7 days';
```

**Target**: 40-60% is strong. < 30% indicates product-market fit issues.

## Scoreboard Calculation

### Automated Weekly Calculation

<Info>
  **Planned Feature**: Cron job to auto-calculate scoreboard metrics every Monday at 2am UTC.
</Info>

**Process**:
1. **Identify cohort**: All signups from previous week (Monday-Sunday)
2. **Calculate WFAU**: Users who performed meaningful action within 7 days of signup
3. **Calculate Day 7 return**: Wait 7 days, then check login activity
4. **Aggregate by source**: Group by `referral_code_used`, UTM parameters, or campaign IDs
5. **Insert/Update**: Upsert into `growth_scoreboard` table

**Example Pseudocode**:
```typescript
async function calculateWeeklyScoreboard() {
  const lastWeekStart = getLastMonday(); // '2026-01-13'
  const sources = ['ambassadors', 'campus_outreach', 'labs', 'organic'];

  for (const source of sources) {
    const signups = await getSignupsBySource(source, lastWeekStart);
    const wfau = await calculateWFAU(signups);
    const day7Returns = await calculateDay7Returns(signups); // May need to wait 7 days

    await upsertScoreboard({
      week_start: lastWeekStart,
      source,
      campus_id: null, // Aggregate across all campuses
      signups: signups.length,
      wfau: wfau.length,
      day_7_return: day7Returns.length,
      day_7_return_pct: (day7Returns.length / wfau.length) * 100
    });
  }
}
```

## Campus-Level Breakdown

### Per-Campus Metrics

**Query**:
```sql
SELECT
  c.name as campus_name,
  gs.source,
  gs.signups,
  gs.wfau,
  gs.day_7_return,
  gs.day_7_return_pct
FROM growth_scoreboard gs
JOIN contexts c ON c.id = gs.campus_id
WHERE gs.week_start = '2026-01-13'
  AND gs.campus_id IS NOT NULL
ORDER BY gs.signups DESC;
```

**Example Output**:
| Campus | Source | Signups | WFAU | D7 Return | D7 % |
|--------|--------|---------|------|-----------|------|
| University of Michigan | campus_outreach | 42 | 39 | 24 | 61.5% |
| UC Berkeley | ambassadors | 35 | 31 | 19 | 61.3% |
| NYU | organic | 18 | 16 | 8 | 50.0% |

**Use Case**: Compare effectiveness of same campaign type across different campuses.

## Platform Split Analytics

### iOS vs Web vs Android

The `platform_split` JSONB field tracks signup distribution:

```json
{
  "ios": 92,        // 50%
  "web": 74,        // 40%
  "android": 19     // 10%
}
```

**Query**:
```sql
SELECT
  source,
  SUM((platform_split->>'ios')::int) as ios_signups,
  SUM((platform_split->>'web')::int) as web_signups,
  SUM((platform_split->>'android')::int) as android_signups
FROM growth_scoreboard
WHERE week_start >= CURRENT_DATE - INTERVAL '4 weeks'
GROUP BY source;
```

**Insights**:
- **iOS dominance**: If iOS > 60%, prioritize iOS features and stability
- **Web growth**: Rising web signups may indicate need for web app improvements
- **Android opportunity**: If Android < 5%, consider if worth maintaining

## Source Performance Analysis

### ROI by Source

<Warning>
  **Budget Tracking**: For paid sources, add cost data to `metadata.budget` to calculate cost-per-signup.
</Warning>

```sql
SELECT
  source,
  SUM(signups) as total_signups,
  SUM(wfau) as total_wfau,
  AVG(day_7_return_pct) as avg_d7_return,
  SUM((metadata->>'budget')::numeric) as total_budget,
  SUM((metadata->>'budget')::numeric) / NULLIF(SUM(signups), 0) as cost_per_signup
FROM growth_scoreboard
WHERE week_start >= CURRENT_DATE - INTERVAL '8 weeks'
GROUP BY source
ORDER BY total_wfau DESC;
```

**Example Output**:
| Source | Signups | WFAU | Avg D7 % | Budget | Cost/Signup |
|--------|---------|------|----------|--------|-------------|
| ambassadors | 348 | 294 | 59.2% | $520 | $1.49 |
| campus_outreach | 208 | 196 | 56.8% | $350 | $1.68 |
| organic | 136 | 112 | 44.1% | $0 | $0 |
| paid | 89 | 67 | 38.5% | $890 | $10.00 |

**Decision**: Ambassadors have best ROI. Paid ads underperforming - optimize or pause.

## Best Practices

<AccordionGroup>
  <Accordion title="Review Weekly, Act on Trends" icon="calendar">
    Check scoreboard every Monday. Don't react to single-week fluctuations - look at 4-week trends.
  </Accordion>

  <Accordion title="Segment by Source AND Campus" icon="filter">
    Aggregate metrics hide insights. Drill down to find what's working where.
  </Accordion>

  <Accordion title="Focus on WFAU, Not Just Signups" icon="bullseye">
    Signups are vanity metric. WFAU = users who got value. Optimize onboarding to maximize WFAU%.
  </Accordion>

  <Accordion title="Improve Day 7 Return Rate" icon="chart-line">
    D7 return predicts long-term retention. Tactics:
    - Send D3/D5 push notifications
    - In-app reminders for upcoming deadlines
    - Weekly digest emails
  </Accordion>

  <Accordion title="Celebrate Wins, Diagnose Drops" icon="party-horn">
    Share big wins with team (e.g., "Best week ever: 250 signups!"). When metrics drop, investigate immediately.
  </Accordion>
</AccordionGroup>

## SQL Reference

**Insert weekly aggregate**:
```sql
INSERT INTO growth_scoreboard (
  week_start, source, signups, wfau, day_7_return, day_7_return_pct
) VALUES (
  '2026-01-13',
  'ambassadors',
  87,
  73,
  44,
  60.3
);
```

**Upsert (update if exists)**:
```sql
INSERT INTO growth_scoreboard (
  week_start, source, campus_id, signups, wfau
) VALUES (
  '2026-01-13', 'organic', NULL, 34, 28
)
ON CONFLICT (week_start, source, campus_id)
DO UPDATE SET
  signups = EXCLUDED.signups,
  wfau = EXCLUDED.wfau,
  updated_at = NOW();
```

**Query 8-week trend**:
```sql
SELECT week_start, source, signups, wfau, day_7_return_pct
FROM growth_scoreboard
WHERE week_start >= CURRENT_DATE - INTERVAL '8 weeks'
  AND campus_id IS NULL -- Aggregate only
ORDER BY week_start DESC, source;
```
