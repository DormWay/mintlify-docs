# AI Insights & Personalization Strategy

**Document Version**: 1.0
**Created**: December 8, 2025
**Purpose**: Comprehensive analysis of data we collect, AI insights we can generate, and how to surface them without being "scary"

---

## Executive Summary

DormWay has access to an unprecedented amount of student data across multiple dimensions:
- **Per-Class**: Syllabus details, grading policies, attendance requirements, professor style
- **Per-Assignment**: Due dates, weights, estimated effort, submission status
- **Per-Student**: Behavioral patterns, energy curves, productivity hours, submission habits
- **Per-University**: LMS type (Canvas/Blackboard/etc.), academic calendar, campus events

This document outlines what insights we can generate and how to surface them in a **non-invasive, trust-building** way.

## Part 1: Data We Currently Store

### 1.1 Syllabus Extraction (Per-Class)

From `syllabus-claude-extractor.ts`, we extract:

### 3.2 Principle: Explain the Data, Not the Surveillance

**Bad (Scary)**:
> "Based on your location data, you have 12 minutes to reach your next class"

**Good (Helpful)**:
> "Your COMM 230 class in BSB1060 starts in 25 minutes. It's a 12-minute walk from the library."

**Principle**: Always explain **what** data informed the insight, not **how** we tracked it.

### 3.3 Friction Points to Consider

| Insight Type | Necessary Friction? | Implementation |
|--------------|---------------------|----------------|
| Attendance warnings | Yes - first time | Explain why we know attendance is graded (from syllabus) |
| Grade decline alerts | Yes - opt-in | "Want grade insights? Connect Canvas" |
| Study time suggestions | No - helpful default | Just show the suggestion |
| Professor style | Optional | Hidden until user asks "tell me about this professor" |
| Workload forecasting | No - helpful default | Show as part of weekly view |

### 3.4 Progressive Disclosure Model

**Level 1: Glanceable** (Always Visible)
- What's due today
- Next class time/location
- Urgent conflicts

**Level 2: Expandable** (Tap to Reveal)
- Why this is prioritized
- Grade weight / impact
- Suggested study blocks

**Level 3: On-Demand** (Ask Ace)
- "Tell me about this professor's style"
- "How am I doing in this class?"
- "What should I prioritize this week?"

## Part 4: University-Specific Detection & Prompts

### 4.1 LMS Detection Strategy

**Step 1: Email Domain Analysis**
```typescript
// When user signs up with university email
async function detectUniversityLMS(email: string): Promise<LMSConfig | null> {
  const domain = email.split('@')[1];

  // Check campus_configs for known mappings
  const campus = await db.campus_configs.findFirst({
    where: { email_domains: { contains: domain } }
  });

  if (campus?.canvas_instance_url) {
    return {
      type: 'canvas',
      instanceUrl: campus.canvas_instance_url,
      features: ['live_events', 'oauth', 'calendar_sync']
    };
  }

  // Fall back to first-user detection
  return null;
}
```

**Step 2: First-User Discovery**
```typescript
// When first user from a university connects Canvas
async function recordLMSDiscovery(userId: string, oauthResponse: CanvasOAuth) {
  const user = await db.accounts.findUnique({ where: { id: userId } });
  const campus = await getCampusFromEmail(user.email);

  if (campus && !campus.canvas_instance_url) {
    // First Canvas user from this campus!
    await db.campus_configs.update({
      where: { id: campus.id },
      data: {
        canvas_instance_url: oauthResponse.instanceUrl,
        lms_detected_at: new Date(),
        lms_detected_by: userId  // Credit the pioneer
      }
    });

    logger.info('LMS discovery', {
      campus: campus.name,
      lmsType: 'canvas',
      detectedBy: userId
    });
  }
}
```

## Part 5: The "Why" Behind Every Insight

### 5.1 Insight Attribution Framework

Every AI insight MUST explain its source:

```typescript
interface InsightWithAttribution {
  insight: string;

  // Attribution (shown on expand/hover)
  source: 'syllabus' | 'canvas' | 'calendar' | 'pattern' | 'campus';
  sourceDetail: string;  // "From your COMM 230 syllabus"

  // Confidence
  confidence: 'certain' | 'likely' | 'suggested';

  // User control
  canDismiss: boolean;
  canDisable: boolean;
}
```

### 5.2 Example Insight Attributions

| Insight | Source | Attribution Text |
|---------|--------|-----------------|
| "Attendance matters" | Syllabus | "From your COMM 230 syllabus: 'Participation includes attendance'" |
| "Assignment worth 25%" | Syllabus | "COMM 230 grading breakdown shows this is 25% of your grade" |
| "You submitted late 3x" | Canvas | "Based on your Canvas submission history" |
| "Start Tuesday 2pm" | Pattern | "You typically complete similar assignments in 3 hours" |
| "12 min walk to class" | Campus | "Based on campus building locations" |

### 6.3 Notification Strategy

| Notification Type | Frequency | Opt-Out |
|------------------|-----------|---------|
| Assignment due (24h) | Per assignment | Per-assignment snooze |
| Class starting (15m) | Per class | Disable per class |
| Grade posted | Immediate | Disable all grade notifications |
| Heavy week warning | Once/week | Disable workload alerts |
| Attendance reminder | First instance | Acknowledge and dismiss |

## Part 7: Privacy & Trust Principles

### 7.1 Core Principles

1. **Explain the data, not the surveillance**
   - Show what we know, not how we learned it

2. **Insights should help, not judge**
   - "Consider starting Tuesday" not "You always procrastinate"

3. **User controls everything**
   - Every insight type can be disabled
   - All data can be deleted

4. **Progressive trust-building**
   - Start with helpful, obvious insights
   - Unlock deeper insights as user engages

5. **Attribution is mandatory**
   - Every insight explains its source

### 7.2 What We DON'T Do

- Never show cross-student comparisons
- Never reveal professor ratings from other students
- Never share location data with other users
- Never push notifications about "missed" activities
- Never make users feel surveilled
