## Executive Summary

The DormWay dashboard (Home V2) is functionally complete but visually plain compared to the polished landing page. This document provides a phased implementation plan to achieve visual parity and enhanced UX, estimated at **10-15 hours total**.

**Key Goals**:
1. Apply glassmorphism and design tokens consistently
2. Make AI insights genuinely useful (explain "why", not just "what")
3. Add micro-animations and interactive feedback
4. Reduce visual noise and improve scannability
5. Create a dashboard that students actually want to look at

---

## Current State Analysis

### What Works Well
- **Modular architecture**: Each widget is self-contained with proper TypeScript types
- **Elastic layouts**: Widgets adapt to different grid sizes (ticker, compact, standard, expanded)
- **Design tokens exist**: `globals.css` has 700+ lines of CSS variables ready to use
- **Shared components**: `shared.tsx` provides reusable patterns (WidgetHeader, MetricCard, etc.)
- **Error boundaries**: Production-safe with graceful degradation

### What Needs Improvement

| Issue | Current State | Target State |
|-------|---------------|--------------|
| **Surface styling** | Solid `bg-[#0A0A0A]` backgrounds | Glassmorphism with backdrop-blur |
| **Shadows** | Basic `shadow-sm` | Multi-layer shadows with depth |
| **Colors** | Hardcoded hex values | CSS variables (`--ds-widget-*`) |
| **AI insights** | "AI-generated" badge only | Explains WHY this is the priority |
| **Typography** | All Inter font | Display font hierarchy |
| **Animations** | Basic transitions | Spring physics, stagger effects |
| **Interactive states** | Minimal hover effects | Rich feedback on interaction |
