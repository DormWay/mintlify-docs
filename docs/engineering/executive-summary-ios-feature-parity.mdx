## 2. What Needs to Be Done

### Priority 0 (Ship Blockers) - 34 Hours Total

#### Feature 1: Canvas Assignments & Grades (16 hours)
**What:** Let students connect Canvas with Personal Access Token, view all assignments with grades

**User Journey:**
1. Student connects Canvas (one-time setup)
2. Opens "Gradebook" tab
3. Sees all assignments with grades (A, B, C, color-coded)
4. Missing/late assignments highlighted in red
5. Tap assignment ‚Üí See details, due date, submission status

**Technical Scope:**
- New service: `CanvasService.swift` (6 API methods)
- New UI: Canvas connection flow + Gradebook view
- Integration: Real-time sync via Ably

**Effort Breakdown:**
- Day 1: Service layer (6h)
- Day 2: UI layer (8h)
- Integration: Ably updates (2h)

---

#### Feature 2: Assignment Completion Tracking (8 hours)
**What:** Let students mark assignments as Started/Complete/Dismissed

**User Journey:**
1. Student sees assignment in gradebook
2. Taps "Start" ‚Üí Creates 60-min focus session
3. Notification after 60 minutes
4. Taps "Complete" ‚Üí Marked done, syncs across devices
5. Or taps "Dismiss" ‚Üí Removes from suggestions

**Why It Matters:**
- Habit formation (students build completion routine)
- Cross-device sync (complete on phone, reflects on web)
- Analytics (track study patterns)

**Technical Scope:**
- New service: `AssignmentCompletionService.swift`
- Update existing UI: Add action buttons to assignment cards
- Integration: Focus timer, Ably sync

**Effort Breakdown:**
- Day 3: Service + UI (7h)
- Integration: Focus timer (1h)

---

#### Feature 3: Notes Sync (10 hours)
**What:** Sync notes across devices with conflict resolution

**User Journey:**
1. Student creates note on iPhone (class notes)
2. Note auto-syncs to server
3. Opens iPad ‚Üí Note appears automatically
4. Edits on iPad ‚Üí Syncs back
5. If edited offline on both devices ‚Üí Conflict resolution UI

**Why It Matters:**
- Students want notes everywhere (phone ‚Üí tablet ‚Üí laptop)
- Current state: Notes siloed to one device (huge pain point)
- Competitive: Most student apps have note sync

**Technical Scope:**
- New service: `NotesSyncService.swift` (SHA-256 conflict detection)
- New UI: Notes list, editor, conflict resolution
- Integration: Auto-sync on background, manual sync button

**Effort Breakdown:**
- Day 4: Service layer (4h)
- Day 5: UI layer (5h)
- Day 6: Testing + polish (1h)

## 9. Open Questions (Need Decisions)

### Product Decisions Required

**Q1: DayPlan Widget**
- iOS has it, but may be hidden via feature flag
- **Decision needed:** Enable by default or keep hidden?
- **Recommendation:** Test with beta users, enable if engagement > 60%

**Q2: Canvas Onboarding Flow**
- **Option A:** Required during onboarding (blocks flow)
- **Option B:** Optional post-onboarding (lower friction)
- **Option C:** Prompt when viewing Academic tab (contextual)
- **Recommendation:** Option C (contextual)

**Q3: Notes Sync Behavior**
- **Option A:** Auto-sync on every change (high battery usage)
- **Option B:** Auto-sync on app background (efficient)
- **Option C:** Manual sync only (user control)
- **Recommendation:** Option B + manual button

### Technical Decisions Required

**Q4: Canvas Caching**
- Web caches for 1 hour
- **Decision:** Same for iOS or more aggressive (2 hours for battery)?
- **Recommendation:** 2-hour cache, manual refresh available

**Q5: Offline Queue Size**
- **Decision:** Max operations to queue when offline?
- **Options:** 50, 100, 200, unlimited
- **Recommendation:** 100 operations, 7-day TTL
