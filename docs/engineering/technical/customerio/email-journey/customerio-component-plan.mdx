# DormWay Customer.io Email Template Plan

## ⚠️ IMPORTANT UPDATE
**We've moved away from Customer.io's EMC component system** due to compatibility issues. This document now describes the **standard liquid template approach**.

## Goals
- Establish a reusable **base layout** so every email automatically carries DormWay brand, spacing, and footer content.
- Provide a library of **liquid template snippets** (hero, stat grid, assignment list, etc.) that can be composed into emails.
- Match the design system from **dormway-lockedin** for brand consistency.
- Use table-based layouts for maximum email client compatibility.

## New Files
- **Base Template**: `customerio-liquid-base-template.html` - Main layout with header/footer
- **Snippets Library**: `liquid-snippets-library.md` - Reusable components

## 1. Design System (from dormway-lockedin)

### Brand Colors
```css
--dormway-purple:       #8b5cf6  /* Primary brand color */
--dormway-purple-light: #a78bfa  /* Hover states */
--dormway-purple-dark:  #6d28d9  /* Pressed states */
--dormway-pink:         #ec4899  /* Accent color */
--dormway-orange:       #f97316  /* Secondary accent */
```

### Background & Surface Colors
```css
--surface-0:      #0f111c      /* Main background */
--surface-1:      rgba(255, 255, 255, 0.03)  /* Card backgrounds */
--surface-2:      rgba(255, 255, 255, 0.05)  /* Elevated surfaces */
--surface-3:      rgba(255, 255, 255, 0.08)  /* Highest elevation */
```

### Text Colors
```css
--text-primary:   #f5f5f5      /* Headings, emphasis */
--text-secondary: #cbd5e1      /* Body text */
--text-tertiary:  #94a3b8      /* Supporting text */
--text-muted:     #64748b      /* De-emphasized text */
```

### Border Colors
```css
--border-subtle:  rgba(255, 255, 255, 0.08)
--border-muted:   rgba(255, 255, 255, 0.12)
--border-strong:  rgba(255, 255, 255, 0.18)
```

### Effects
- **Glass effect**: `backdrop-filter: blur(16px); background: rgba(255, 255, 255, 0.03);`
- **Gradient**: `linear-gradient(135deg, #8b5cf6 0%, #ec4899 50%, #f97316 100%)`
- **Shadow**: `0 8px 32px rgba(0, 0, 0, 0.2)`

## 2. Base Layout Template
Create a layout in Customer.io (Content → Layouts) and paste the content from `customerio-liquid-base-template.html`. Set it as the default for DormWay campaigns. This base layout provides consistent branding, spacing, and footer across all emails. Content is inserted via the `{{{ content }}}` placeholder.

**Key Features:**
- **Dark-first design** matching dormway-lockedin
- **Glassmorphic effects** with gradient background
- **600px max width** for optimal email client compatibility
- **Table-based layout** for maximum compatibility
- **Gradient brand logo** (purple → pink → orange)
- **Social media icons** in footer
- **Legal footer** with unsubscribe and preferences links
- **Responsive design** with mobile breakpoints
- All email content goes in the `{{{ content }}}` area

## 3. Reusable HTML/Liquid Code Blocks

⚠️ **IMPORTANT**: Customer.io snippets **cannot reference message body variables**. Instead, copy HTML directly from `liquid-snippets-library.md` into your message body.

**What works:**
- ✅ Pasting HTML directly into message body
- ✅ Using Customer.io attributes: `{{customer.first_name}}`
- ✅ Using event data: `{{event.course_name}}`
- ✅ Conditional logic and loops with Customer.io data

**What doesn't work:**
- ❌ Assigning variables in message, then referencing in snippets
- ❌ Creating reusable snippet components with parameters

Use the code blocks from `liquid-snippets-library.md` as **copy-paste templates**, not as actual Customer.io snippets.

### 3.1 Hero Section
**Gradient Hero** - Copy the HTML from `liquid-snippets-library.md` and customize the text:
```html
<table role="presentation" width="100%">
    <tr>
        <td style="padding: 40px 32px; background: linear-gradient(135deg, #8b5cf6 0%, #7c3aed 50%, #6d28d9 100%); border-radius: 12px;">
            <h1 style="font-size: 32px; color: #ffffff;">Welcome to DormWay</h1>
            <p style="font-size: 16px; color: rgba(255, 255, 255, 0.9);">The OS for College Life</p>
        </td>
    </tr>
</table>
```

**Using Customer.io data:**
```html
<h1>Welcome {{customer.first_name}}</h1>
<p>You're at {{customer.campus_name}}</p>
```

### 3.2 Stats Grid
Three-column layout with metrics - Copy from library and replace with Customer.io attributes:

**Using event data:**
```html
<h2>{{event.assignment_count}}</h2>
<p>assignments</p>
```

**Using customer attributes:**
```html
<h2>{{customer.gpa}}</h2>
<p>current GPA</p>
```

See `liquid-snippets-library.md` for complete table-based HTML structure.

### 3.3 Assignment/Event List

```html
        </td>
    </tr>
</table>
{% endfor %}
```

See `liquid-snippets-library.md` for complete HTML with priority badges and styling.

## 6. Migration from EMC Components

**Why we moved away from EMC:**
- Limited browser/client support
- Inconsistent rendering across email clients
- Complex debugging and preview issues
- Limited customization options

**Benefits of liquid templates:**
- ✅ Better email client compatibility
- ✅ Standard HTML table layouts (proven email pattern)
- ✅ Full control over styling
- ✅ Easier to debug and maintain
- ✅ Matches dormway-lockedin design system exactly
