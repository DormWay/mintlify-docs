# DormWay Customer.io Liquid Snippets Library

This document provides reusable HTML/Liquid snippets for building emails using Customer.io's standard liquid template system. These snippets match the dormway-lockedin design system.

## ‚ö†Ô∏è IMPORTANT: Customer.io Snippet Limitations

**Snippets CANNOT reference variables from the message body!**

Customer.io snippets have their own scope and cannot access variables you assign in the message. This means:

‚ùå **This DOES NOT work:**
```liquid
<!-- In message body -->
{% assign hero_title = "Welcome to DormWay" %}
{{snippets.hero_gradient}}  <!-- Can't see hero_title! -->
```

‚úÖ **This WORKS:**
```liquid
<!-- In message body - just paste the HTML directly -->
<table role="presentation">
  <tr><td>Welcome to DormWay</td></tr>
</table>
```

‚úÖ **This ALSO works (using Customer.io attributes):**
```liquid
<!-- In snippet - uses Customer.io data -->
<h1>Welcome {{customer.first_name}}</h1>
```

### Recommended Approach
1. **Use layouts** for base template (header/footer) ‚úÖ
2. **Paste HTML directly** into message body (copy from this library)
3. **Use Customer.io attributes** like `{{customer.first_name}}`, `{{event.course_name}}`
4. **Don't create snippets** unless they're fully self-contained

1. **Create Base Layout**: Upload `customerio-liquid-base-template.html` as a layout in Customer.io

4. **Test**: Preview in Customer.io and send test emails to verify rendering

## Tips for Email Development

1. **Inline all CSS** - Email clients strip `<style>` tags
2. **Use tables for layout** - Flexbox/Grid don't work in email
3. **Test in multiple clients** - Use Litmus or Email on Acid
4. **Keep it simple** - Complex layouts break in email
5. **Use web-safe fonts** - Arial, Helvetica, Georgia, Times
6. **Alt text for images** - Essential for accessibility
7. **UTM parameters** - Track link performance
8. **Mobile-first** - 60%+ of emails opened on mobile

## Customer.io Data Reference

### Available Variables

**Customer Attributes** (set via identify or API):
```liquid
{{customer.id}}
{{customer.email}}
{{customer.first_name}}
{{customer.last_name}}
{{customer.campus_name}}
{{customer.campus_id}}
{{customer.created_at}}
```

**Event Data** (passed when triggering campaign):
```liquid
{{event.course_name}}
{{event.course_code}}
{{event.assignment_count}}
{{event.assignments}}  <!-- Array -->
```

**Built-in Variables**:
```liquid
{{unsubscribe_url}}      <!-- Required in footer -->
{{preferences_url}}       <!-- Preference center -->
{{reply_to_email}}
```

**Conditional Logic**:
```liquid
{% if customer.campus_name %}
  <p>Welcome {{customer.campus_name}} student!</p>
{% else %}
  <p>Welcome to DormWay!</p>
{% endif %}
```

**Loops**:
```liquid
{% for item in event.assignments %}
  <p>{{forloop.index}}. {{item.name}}</p>
{% endfor %}
```

**Filters**:
```liquid
{{customer.first_name | capitalize}}
{{event.created_at | date: "%B %d, %Y"}}
{{event.price | money}}
```

### Sending Event Data from Backend

```typescript
// TypeScript example - triggering transactional email with data
import { CustomerIO } from 'customerio-node';

const cio = new CustomerIO(siteId, apiKey);

await cio.sendEmail({
  transactional_message_id: 'syllabus-processed',
  to: user.email,
  identifiers: { id: user.id },
  message_data: {
    course_name: 'Introduction to Computer Science',
    course_code: 'CS 101',
    assignment_count: 12,
    assignments: [
      { name: 'Problem Set 1', due_date: 'Oct 15', priority: 'high' },
      { name: 'Lab Report', due_date: 'Oct 20', priority: 'medium' }
    ]
  }
});
```
