## Page Structure

The landing page (`src/app/page.tsx`) was simplified in December 2025 (DORM-812) from 13+ sections to 9 focused sections. The live demo iframe was removed, and the PLG upload became the centerpiece.

### Current Structure (9 Sections)

#### 1. **Hero Section** (`HeroSection.tsx`)
- Headline: "Your college life. One dashboard."
- Proof points: Free forever, Works at 6,134+ schools, Beta available now
- Primary CTA button
- School detection and personalization
- Subtle ambient glow background (no iframe demo)

#### 2. **PLG Upload Section** (`BootstrapSection.tsx`)
- Email input + 3 file drop zones (schedule, syllabus, other)
- Immediate value proposition - upload files to get started
- Subtle radial gradient glow (design-system aligned)
- Uses `useMultiFileUpload` hook and `QuickStartUpload` component

#### 3. **Problem Section** (`ProblemSection.tsx`)
- "Students juggle 12+ apps weekly" messaging
- SystemsCloud visualization showing app chaos
- Side-by-side: scattered apps vs. unified DormWay dashboard
- Research stats from EDUCAUSE studies

#### 4. **Term Preview Section** (`TermPreviewSection.tsx`)
- "What You'll Get" - interactive demo data preview
- Animated counters: courses detected, assignments found, conflicts flagged
- Interactive workload chart (click/hover weeks)
- Expandable course list preview
- Two-column layout for visual balance

#### 5. **Jobs To Be Done Section** (`JobsToBeDoneSection.tsx`)
- "See How It Works" - 4 visual feature cards
- Always know what's due (dashboard mockup)
- See what matters today (priority view)
- Catch problems early (alert preview)
- One place, not 12 apps (unified sources)

#### 6. **Comparison Section** (`ComparisonSection.tsx`)
- "Why not just use Canvas/Notion/To-Do/ChatGPT?"
- Rotating comparison between competitors and DormWay
- Side-by-side columns: competitor limitations vs DormWay solutions
- Fixed height containers to prevent layout shifts
- Auto-rotation with manual navigation controls

#### 7. **Ace Section** (`AceSection.tsx`)
- "Meet Ace" - AI assistant introduction
- 3-column Q&A examples (policies, deadlines, grading)
- Interactive card selection with source citations
- "Try asking" prompt suggestions at bottom

#### 8. **Mobile App Section** (`MobileAppSection.tsx`)
- iOS app coming soon preview
- Waitlist signup CTA
- Feature highlights

#### 9. **FAQ + Footer CTA**
- `FAQSection.tsx` - Common questions with expandable answers
- Bottom CTA - Final call-to-action with PrimaryCTA
- `Footer.tsx` - Links, legal, social

### Removed Sections (DORM-812)
The following were removed during the December 2025 simplification:
- Live demo iframe (was in HeroSection)
- StickyMiniNav
- GetStartedSection (merged into BootstrapSection)
- HowItWorksWithUpload (replaced by BootstrapSection)
- SocialProofSection
- StudentScenariosSection
- FounderSection ("Who Made This")
- CollapsibleBlogSection

## Theme System

The landing page supports multiple themes:

### Available Themes

1. **System** - Respects user's OS preference
2. **Light** - Light mode
3. **Dark** - Dark mode (default for unauthenticated users)
4. **Glass** - Glassmorphic dark mode

### Theme Configuration

- **File:** `src/lib/theme-config.ts`
- **Context:** `src/contexts/ThemeContext.tsx`
- **Utilities:** `src/lib/theme-utils.ts`
- **DOM Manipulation:** `src/lib/theme-dom.ts`

### Theme Bootstrap

The theme system is bootstrapped in `src/app/layout.tsx` with:
- Theme configuration data
- Design tokens
- Accent presets
- Font sizes
- Storage sanitization

### CSS Files

1. **globals.css** - Global styles and theme overrides
2. **theme-animations.css** - Theme transition animations
3. **glass-theme.css** - Glass theme specific styles
4. **academic-animations.css** - Academic component animations
5. **light-theme-fixes.css** - Light theme fixes and overrides

## Performance Considerations

1. **Lazy Loading** - Below-fold sections are lazy loaded
2. **Code Splitting** - Components split into separate chunks
3. **Suspense Boundaries** - Loading states for async components
4. **Reduced Motion** - Respects user preferences
5. **Intersection Observer** - Pauses animations when off-screen

## Summary

The DormWay landing page is a streamlined, performant, and accessible single-page application featuring:

- **9 focused landing page sections** (simplified from 13+ in DORM-812)
- **PLG-first approach** - upload section at position #2
- **10+ active components** with legacy components preserved
- **4 theme modes** (system, light, dark, glass)
- **Ambient background** with CSS radial gradients (no WebGL)
- **School detection and personalization**
- **Multi-file upload** functionality via `useMultiFileUpload` hook
- **Comprehensive design system** with `--ds-*` CSS variables
- **Performance optimizations** (lazy loading, code splitting, Suspense boundaries)
- **Accessibility features** (reduced motion, keyboard navigation)
- **Stable layouts** - fixed height containers in ComparisonSection to prevent redraws

### Recent Changes (DORM-812, December 2025)
- Removed live demo iframe from hero
- Added TermPreviewSection with animated counters
- Enhanced AceSection with 3-column Q&A and source citations
- Refined BootstrapSection styling to match design system
- Stabilized ComparisonSection height to prevent layout shifts
- Removed 4+ unused sections (FounderSection, SocialProof, etc.)
