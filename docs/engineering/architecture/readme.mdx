This is distinct from [Technical Architecture](/docs/engineering/ai-crews/readme), which focuses on implementation patterns and "HOW we build it."

## Architectural Decision Records (ADRs)

### What are ADRs?
ADRs document significant architectural decisions, including:
- **Context**: Why the decision was needed
- **Decision**: What was decided
- **Consequences**: Trade-offs and implications
- **Alternatives**: Other options considered

## Distinction: System vs Technical Architecture

### System Architecture (This Section)
- **Focus**: Decisions and design
- **Audience**: Architects, tech leads, senior engineers
- **Questions**: WHAT are we building? WHY this approach?
- **Examples**: "Why REST over GraphQL?", "Why Ably for real-time?"

### Technical Architecture ([Technical Architecture](/docs/engineering/ai-crews/readme))
- **Focus**: Implementation patterns
- **Audience**: All engineers
- **Questions**: HOW do we implement this?
- **Examples**: "How do we handle errors?", "How do job processing flows work?"

## Architecture Principles

DormWay architecture follows these principles:
1. **Simplicity over Cleverness**: Choose simple, well-understood patterns
2. **Incremental Migration**: New systems coexist with old during transitions
3. **Leverage Standards**: Use HTTP caching, standard REST, proven patterns
4. **Document Decisions**: All major decisions have ADRs
5. **Evolution over Perfection**: Architecture evolves based on real needs

## Architecture Review Process

### When to Create Architecture Docs
- **Major system changes**: New services, significant refactors
- **Technology choices**: Adopting new frameworks, databases, services
- **Integration decisions**: Adding external service integrations
- **Performance changes**: Caching strategies, data flow optimizations

## Deprecated Architectures

### Handling Deprecation
When an architecture is superseded:
1. Prefix filename with "DEPRECATED -"
2. Update frontmatter: `status: deprecated`
3. Add "Replaced By" section linking to new architecture
4. Move to [Archive](/docs/engineering/ai-crews/readme) after 6 months

### Why Keep Deprecated Docs?
- Institutional knowledge
- Migration context
- Understanding evolution
- Debugging legacy issues
