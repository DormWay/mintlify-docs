# 5 Core User Journeys

These are the ONLY user experiences shipping in the [Launch Plan](/docs/plans/planning/launch-plan) for [DormWay](/docs/operations/meta/obsidian-indexes/dormway). Everything else is post-launch.

## 1. First-Time [Onboarding](/docs/product/features/onboarding/onboarding)

**Flow**: App Store → Install → Account Creation → Profile Setup → Permissions → Home Screen

**Critical Elements**:
- [Privacy Consent Default](/docs/product/features/student-experience/privacy-consent-default) must be false
- Campus Selection must work
- Major Selection must work
- Smooth transition to home

**Links to**: [Authentication](/docs/engineering/technical/authentication/readme), User Profile, Permissions System

## 2. Syllabus Loading

**Flow**: Upload Button → File Selection → Processing → Results Display

**Critical Elements**:
- PDF support (working)
- Word Document Extraction (needs implementation)
- Processing feedback
- BrainGains integration

**Links to**: [Document Processing](/docs/engineering/technical/ai/document-processing), [AI System](/docs/engineering/technical/ai/readme), File Upload

## 3. [Calendar Connection](/docs/product/features/calendar-system)

**Flow**: Permission Request → [Calendar Selection](/docs/engineering/technical/calendar/calendar-selection-feature) → Sync → Display

**Critical Elements**:
- [Calendar Selection](/docs/engineering/technical/calendar/calendar-selection-feature) UI (completed)
- [Timezone Conversions](/docs/engineering/technical/data-and-time/timezone-conversions) (needs fixing)
- [Event Lifecycle](/docs/engineering/technical/calendar/calendar-event-removal-and-lifecycle-management) management
- Preference persistence

**Links to**: [EventKit Integration](/docs/engineering/technical/ios/ios-calendar-sync-system), [Calendar Preferences](/docs/engineering/technical/calendar/calendar-selection-feature), [Time Blocks](/docs/engineering/technical/calendar/schedules-time-blocks-and-reconciliation)

## 4. First BrainGains Interaction

**Flow**: Question Input → RAG Retrieval → Answer Generation → Display

**Critical Elements**:
- Syllabus must be loaded first
- Quick response time
- Accurate answers
- Source citations

**Links to**: [AI System](/docs/engineering/technical/ai/readme), RAG Infrastructure, [Vectara Integration](/docs/engineering/technical/integrations/vectara-integration)

## 5. Daily Return User

**Flow**: App Launch → Context Check → Dashboard Update → Navigation

**Critical Elements**:
- Fast launch
- Fresh data via [Ably Integration](/docs/engineering/technical/integrations/ably-integration)
- Context-appropriate display
- Push Notifications if enabled

**Links to**: [Context System](/docs/engineering/architecture/sot-context-system), Real-time Updates, Dashboard

## Out of Scope

Everything not listed above, including:
- Study groups
- Grade tracking
- Dining features
- Social features
- Transportation
- Desktop/web apps
