## Journey 2: Syllabus Loading ‚úÖ

**Flow**: Email ‚Üí Webhook ‚Üí S3 ‚Üí Parallel Processing ‚Üí Results ‚Üí Web Viewer ‚Üí Email

### Current Implementation (Production)
- **Email Processing**: syllabus@dormway.app via Postmark
- **Multi-Attachment**: Parallel Temporal workflows (1 per file)
- **Storage**: S3 with presigned URLs
- **Formats**: PDF ‚úÖ, TXT ‚úÖ, DOCX ‚úÖ, Word (improving)
- **Web Viewer**: syllabus.dormway.app with CTAs
- **Notifications**: Customer.io with "DormWay" sender

### Processing Pipeline
1. Email to syllabus@dormway.app
2. Postmark webhook ‚Üí API Router
3. Parallel S3 uploads (each attachment)
4. Spawn Temporal workflow per attachment
5. AI Crews extract 80+ data points
6. Store in PostgreSQL service_data
7. Send email with web link
8. Display at syllabus.dormway.app/[token]

### iOS 26 Enhancements
- **Foundation Models**: Instant on-device syllabus parsing
- **No Network Needed**: Complete offline processing
- **Privacy First**: Syllabus never leaves device unless user chooses
- **Hybrid Mode**: Complex analysis via server, simple on device

**Status**: ‚úÖ COMPLETE - Email-to-syllabus pipeline fully operational

## Multi-Campus Scaling

### Campus Registry Infrastructure ‚úÖ
**Already Implemented**: Complete registry of 12,506 campuses in `campus_registry` table
- **Coverage**: Every US college and university
- **Geolocation**: Latitude/longitude for weather and local services
- **Metadata**: Enrollment, type (public/private), aliases
- **Onboarding**: Campus selector UI connected to registry

### Campus Data Model
```sql
-- Production campus_registry table
id: UUID
name: "University of Michigan"
domain: "umich.edu"
city: "Ann Arbor"
state: "Michigan"
latitude: "42.278044"
longitude: "-83.738220"
enrollment_total: 47907
type: "Public"
```

### Campus-Specific Services
```typescript
interface CampusContext {
  campusId: string;  // From campus_registry
  location: {
    lat: number;     // From campus_registry
    lng: number;     // From campus_registry
  };
  services: {
    weather: WeatherService;      // Uses lat/lng
    news: CampusNewsService;       // Domain-specific
    events: EventsService;         // Campus calendar
    dining: DiningService;         // If available
    transit: LocalTransitService;  // City-specific
  };
}
```

### Scaling Strategy
- **Onboarding**: User selects from 12,506 campuses
- **Geofencing**: Auto-configured using campus lat/lng
- **Weather**: Location-based using coordinates
- **Local Services**: City/state specific APIs
- **Progressive Enhancement**: Features enable based on campus
