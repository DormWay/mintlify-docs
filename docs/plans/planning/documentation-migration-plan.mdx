# Documentation Migration Plan

## üéØ Migration Goals

1. **Preserve existing knowledge** - No information loss
2. **Enhance discoverability** - Better search and linking
3. **Improve workflow** - Faster documentation creation
4. **Enable collaboration** - Multi-user editing with Git sync

## üìä Current State Analysis

### MkDocs Content (~/dormway/docs/docs/)
- **Features**: ~50+ ideation and implementation docs
- **Worklogs**: Daily development logs
- **Architecture**: System design documents
- **Backend/Mobile**: Technical documentation
- **Meetings**: Meeting notes and decisions

### Content Types & Migration Priority

| Content Type | Count | Priority | Migration Approach |
|-------------|-------|----------|-------------------|
| Work Logs | ~30 | High | Bulk convert with script |
| Features | ~50 | High | Semi-automated + cleanup |
| Architecture | ~20 | Medium | Manual with enhanced linking |
| API Docs | ~15 | Low | Keep in MkDocs? |
| Meeting Notes | ~10 | Medium | Quick conversion |

## üîÑ Migration Process

### Phase 1: Automated Conversion (Week 1)
1. **Create conversion script**
   ```bash
   # Convert markdown files preserving structure
   # Update internal links from /docs/ to Obsidian format
   # Extract and convert frontmatter
   ```

2. **Bulk migrate worklogs**
   - Maintain date-based naming
   - Convert links to Obsidian format
   - Preserve Linear ticket references

3. **Test with subset**
   - Migrate 5 docs manually
   - Verify links work
   - Check formatting

### Phase 2: Content Enhancement (Week 2)
1. **Add bidirectional links**
   - Link features to implementations
   - Connect worklogs to features
   - Create topic maps

2. **Build specialized views**
   - Feature status dashboard
   - Sprint progress tracker
   - Architecture overview canvas

3. **Create index notes**
   - Main categories
   - Tag taxonomies
   - Navigation guides

### Phase 3: Workflow Integration (Week 3)
1. **Team onboarding**
   - Train on Obsidian basics
   - Establish conventions
   - Document workflows

2. **Automation setup**
   - Daily note templates
   - Meeting note templates
   - Linear integration?

3. **Deprecate MkDocs?**
   - Or maintain for public docs
   - Decision point
