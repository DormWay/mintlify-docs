3. **AI/ML reorg is legit** - From 5 scattered locations to 2 organized

### Where This Might Bite Later âš ï¸

#### 1. READMEs Are Too Verbose / Time-Bound
**Problem**: READMEs include:
- Specific stats ("Total docs: 43", "Active: 39")
- Dates ("Last Updated: 2025-11-16")
- Health score commentary
- Implementation details

**Why it's bad**: These will drift and become lies.

**Solution**:
- **Keep in READMEs** (evergreen): Purpose, structure, key docs, related sections
- **Move to frontmatter or Meta** (can drift): File counts, dates, health scores, stats
- Push time-bound content to `Meta/Documentation/OBSIDIAN_DOCS_STATUS.md`

#### 2. Cross-Links Will Break
**Problem**: Many `[Intentional Missing Links](/docs/operations/meta/intentional-missing-links)` between sections. Renames will break them silently.

**Solution**:
- Quarterly: Run Obsidian "Check for broken links"
- When renaming: Use Obsidian's built-in rename (auto-updates links)
- Add to maintenance runbook

#### 3. Meta Language vs Real Docs
**Problem**: Phase summaries talk about "health scores", "execution time", "Phase 1.1" - this is project management content, not system documentation.

**Solution**:
- Move to `Meta/Documentation/` folder
- Keep only practical guides at root (like OBSIDIAN_QUICK_START.md)
