## Weekly Tasks (5 minutes)

**Frequency**: Every Friday or end of sprint

### Link Health Check
- [ ] Run entry scope: `node scripts/obsidian-tools/check-links.js --scope=entry --top=25`
- [ ] Run canonical scope: `node scripts/obsidian-tools/check-links.js --scope=canonical --top=25`
- [ ] Run verification gate (strict for Operations/Runbooks): `node scripts/obsidian-tools/verify-current.js --scope=canonical --require-verified-path=Operations/Runbooks --fail-on=tbd,wip --warn-on=todo`
- [ ] If entry scope has any missing links, fix immediately (no placeholders allowed).
- [ ] For canonical scope, follow the decision rules below.

### Suggested CI / Weekly Commands
- Entry gate (must be clean):
  - `node scripts/obsidian-tools/check-links.js --scope=entry --fail-on=entry`
- Canonical gate (ignore aspirational hubs):
  - `node scripts/obsidian-tools/check-links.js --scope=canonical --fail-on=canonical --ignore-file="Operations/Meta/DormWay Documentation Wiki.md" --gate-status=current`
- Canonical triage helper:
  - `node scripts/obsidian-tools/check-links.js --scope=canonical --report-untracked=25 --ignore-file="Operations/Meta/DormWay Documentation Wiki.md" --gate-status=current`

Flags:
- `--ignore-path=PATH` ignores a folder prefix for gating and untracked reporting.
- `--ignore-file=REL_PATH` ignores a specific file for gating and untracked reporting.
- `--append-intentional=N` appends top N untracked targets to the intentional tracker.
- `--gate-status=STATUS` limits canonical gating to docs with matching frontmatter status (e.g., `current`).

### Review Recent Changes
- [ ] Check files created/edited this week (use Obsidian recent files or git log)
- [ ] Verify new files have proper frontmatter (type, status, owner, tags)
- [ ] Ensure new files link from/to relevant section READMEs
- [ ] Check that cross-references are bidirectional where appropriate
- [ ] Only promote to `status: current` after verifying the doc against code/config; otherwise keep `status: draft` and add a short "Verification Needed" section.

### Archive Completed Work
- [ ] Move completed workplans to Archive/Workplans/
- [ ] Update feature status from "In Development" to "Active" upon launch
- [ ] Archive deprecated feature docs with redirect notices

### Quick Health Check
- [ ] Scan Daily Notes/ for any docs that should be Session Notes in Memory/
- [ ] Check if any large files (>1000 lines) should be split or archived
- [ ] Verify Templates/ haven't been modified accidentally

**Time Required**: ~5 minutes

## Link Triage Decision Rules

When a canonical link is missing:
- **Map-to-existing-doc**: use `scripts/obsidian-tools/map-missing-links.js` if the doc exists under a new name/path.
- **Create-anchor-doc**: only if the target appears multiple times and is clearly canonical (not a one-off idea).
- **Intentional placeholder**: log it in `DormWay/Operations/Meta/Intentional Missing Links.md` with owner + reason.

Do not create stubs for ticket IDs or one-off phrases.

## Monthly Tasks (30 minutes)

**Frequency**: First Monday of each month

### Content Staleness Review
- [ ] Generate report of files not modified in 60+ days
  - Use: `find obsidian-vault/DormWay -name "*.md" -mtime +60`
- [ ] Review each stale file for:
  - Still accurate?
  - Should be archived?
  - Needs deprecation notice?
  - Owner still correct?

### README Accuracy Check
- [ ] Review README.md files in major sections
- [ ] Update stats if significantly drifted (optional - can move to frontmatter)
- [ ] Ensure "Related Sections" links are current
- [ ] Check that "Key Documents" lists are relevant

### Duplicate Content Scan
- [ ] Search for common terms to find duplicate docs
- [ ] Merge duplicates or add cross-references
- [ ] Archive older/inferior versions

### Tag Hygiene
- [ ] Review tag usage across vault
- [ ] Consolidate similar tags (e.g., `#ios` vs `#iOS` vs `#mobile`)
- [ ] Update Tag glossary if one exists

**Time Required**: ~30 minutes

## Quarterly Tasks (2 hours)

**Frequency**: End of each quarter (March, June, September, December)

### Comprehensive Link Check
- [ ] Run Obsidian "Check for broken links" plugin
- [ ] Fix all broken wiki links `Like This`
- [ ] Update links to renamed/moved files
- [ ] Remove links to deleted files (or restore if needed)

**How to check**:
1. Open Obsidian
2. Command palette (Cmd+P)
3. Type "Check for broken links"
4. Fix each broken link

### README Deep Review
- [ ] Read through all major section README.md files
- [ ] Verify structure descriptions match reality
- [ ] Update "Key Documents" lists
- [ ] Check that entry points guide users effectively
- [ ] Remove time-bound stats (optional)

### Vault Health Metrics
- [ ] Run vault export script
- [ ] Calculate file counts per section
- [ ] Identify largest files (candidates for splitting/archiving)
- [ ] Check growth rate (files added per month)
- [ ] Update Index.md with current stats

```bash
# Export vault
.repos/dormway-platform/scripts/obsidian/export_obsidian_vault.sh \
  /path/to/obsidian-vault/DormWay \
  /path/to/OBSIDIAN_VAULT_EXPORT.txt

# Check stats
wc -l OBSIDIAN_VAULT_EXPORT.txt
find obsidian-vault/DormWay -name "*.md" | wc -l
```

### Archive Old Content
- [ ] Identify deprecated content >6 months old
- [ ] Move to Archive/ with appropriate subdirectories
- [ ] Add redirect notices in original locations
- [ ] Update links to archived content

### Template Review
- [ ] Review Templates/ for any needed updates
- [ ] Check if new doc patterns have emerged
- [ ] Update templates to reflect current best practices
- [ ] Ensure template examples are accurate

**Time Required**: ~2 hours

## Annual Tasks (4 hours)

**Frequency**: End of calendar year (December)

### Full Vault Audit
- [ ] Review entire vault structure for major inefficiencies
- [ ] Identify sections that have grown too large
- [ ] Consider reorganization if needed (with team buy-in)
- [ ] Archive all deprecated content >1 year old

### Archive Consolidation
- [ ] Consolidate Archive/ by year
  - `Archive/2024/`, `Archive/2025/`, etc.
- [ ] Create Archive/README with index of archived content
- [ ] Remove truly obsolete content (with backup)

### README Refresh
- [ ] Update all major section READMEs
- [ ] Rewrite any stale introductions
- [ ] Ensure READMEs reflect current reality
- [ ] Add/remove sections as needed

### Naming Convention Review
- [ ] Review actual file naming patterns
- [ ] Update Meta/Documentation-Standards.md if patterns have evolved
- [ ] Consider renaming files for consistency (use Obsidian rename!)
- [ ] Document any new conventions

### Celebration
- [ ] Review year-over-year improvement
- [ ] Document wins (better navigation, clearer structure, etc.)
- [ ] Plan next year's improvements
- [ ] Share documentation improvements with team

**Time Required**: ~4 hours

### Obsidian Features
- **Graph View**: Visualize vault connections
- **Backlinks**: See what links to current doc
- **Unlinked Mentions**: Find potential links
- **Search**: Full-text search across vault
- **Rename**: Auto-updates all wiki links

  /Users/rileycrimmins/DormWay/dormway-beta-workspace/obsidian-vault/DormWay \

Generates:
- `OBSIDIAN_VAULT_EXPORT.txt` - Full export
- `OBSIDIAN_VAULT_EXPORT_outline.md` - Heading structure
- `OBSIDIAN_VAULT_EXPORT_key_sections.md` - Key sections only
- `OBSIDIAN_VAULT_EXPORT_file_index.csv` - File metrics

### Search Commands
```bash
# Find files modified in last 60 days
find obsidian-vault/DormWay -name "*.md" -mtime -60

# Find files NOT modified in last 60 days
find obsidian-vault/DormWay -name "*.md" -mtime +60

# Count files per directory
find obsidian-vault/DormWay -type d -exec sh -c 'echo "$(find "$1" -maxdepth 1 -name "*.md" | wc -l) $1"' _ {} \;

# Find large files (>100KB)
find obsidian-vault/DormWay -name "*.md" -size +100k

# Find files with no frontmatter
grep -L "^---" obsidian-vault/DormWay/**/*.md
```

## Key Principles

1. **Incremental > Big Bang**
   - Fix as you touch things
   - Don't force massive cleanup sessions
   - Small consistent effort compounds

2. **Automation Where Possible**
   - Use Obsidian's built-in tools
   - Leverage export scripts
   - Find commands for common tasks

3. **Prevention > Cure**
   - Use templates for consistency
   - Educate team on best practices
   - Make good habits easy

4. **Archive, Don't Delete**
   - Preserve institutional knowledge
   - You never know when you'll need it
   - Disk space is cheap, lost knowledge is expensive

5. **Link Hygiene**
   - Use Obsidian rename (never manual)
   - Check quarterly for broken links
   - Cross-link liberally but meaningfully
