   - Trust this vault

## ğŸš¨ Important: Git Integration

The `obsidian-git` plugin needs configuration:

1. **Set up authentication**
   - Use SSH keys or GitHub token
   - Configure in plugin settings

2. **Set commit message**
   ```

   ```

3. **Auto-backup interval**
   - Recommended: 10 minutes
   - Auto-pull on startup: Yes

When plugins are added/updated:

1. **Adding a new plugin**:
   ```bash
   # After installing in Obsidian
   git add .obsidian/community-plugins.json
   git add .obsidian/plugins/new-plugin-name/
   git commit -m "Add [plugin-name] to vault"
   ```

2. **Updating plugin settings**:
   ```bash
   # Settings auto-save to .obsidian/plugins/
   git add .obsidian/plugins/
   git commit -m "Update [plugin-name] settings"
   ```

3. **Other team members**:
   ```bash
   git pull
   # Open Obsidian
   # Install any new plugins shown in settings
   ```

## ğŸ› ï¸ Troubleshooting

### Plugin not loading settings?
1. Check if `.obsidian/plugins/[plugin-name]/` exists
2. Restart Obsidian
3. Reinstall the plugin

### "Failed to list templates: no template folder"
This means Templater's configuration file is missing. Fix:
1. Create directory: `.obsidian/plugins/templater-obsidian/`
2. Add `data.json` with template folder config
3. Restart Obsidian
4. The config is now tracked in Git for future users

### Git conflicts in .obsidian?
Usually safe to accept incoming changes for:
- `workspace.json` (just layout)
- `appearance.json` (theme preferences)

Be careful with:
- Plugin data.json files (may contain API keys)
- `app.json` (core settings)
