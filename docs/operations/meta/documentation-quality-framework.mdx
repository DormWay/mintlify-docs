### 5. Duplicate/Overlapping Content

**Problem**: Same topic documented in multiple places, creates confusion.

**How to Find**:
```bash
# Search for common topics across vault
# Example: "authentication"
grep -r "authentication" obsidian-vault/DormWay --include="*.md" -l | sort

# Look for similar file names
find obsidian-vault/DormWay -name "*auth*.md"
find obsidian-vault/DormWay -name "*widget*.md"
```

**Fix Priority**: ‚≠ê‚≠ê MEDIUM

**Action Steps**:

**For each duplicate, choose canonical version**:
1. **Identify**: Find all docs on same topic
2. **Compare**: Which is most complete/accurate?
3. **Merge**: Consolidate into canonical version
4. **Redirect**: Add redirect notices in old locations
5. **Archive**: Move old versions to Archive/

**Example**:
```markdown
Found:
- Architecture/Widget-System.md (comprehensive)
- Features/Widgets/Overview.md (outdated)
- Technical/iOS/Widget-Integration.md (implementation)

Decision:
- Keep: Architecture/Widget-System.md (system design)
- Keep: Technical/iOS/Widget-Integration.md (implementation)
- Archive: Features/Widgets/Overview.md (outdated, redundant)
- Add cross-links between Architecture and Technical docs
```

**Time Estimate**: 4-6 hours (depends on extent)

### 6. Unclear Ownership (No owner field)

**Problem**: No one responsible = docs drift and become stale.

**How to Find**:
```bash
# Find docs without owner field
grep -L "^owner:" obsidian-vault/DormWay/**/*.md

# Or search in Obsidian for missing "owner:" field
```

**Fix Priority**: ‚≠ê LOW (but important for maintenance)

**Action Steps**:
1. Add `owner:` field to frontmatter
2. Assign based on:
   - Current maintainer of feature
   - Team responsible for area (Platform, iOS, AI, Design)
   - Original author if still relevant

**Owner Guidelines**:
- **Team names** for broad areas: `Platform`, `iOS`, `AI`, `Design`
- **Person names** for specific features/components
- **Update** when ownership transfers

**Time Estimate**: 1-2 hours

### 7. Orphaned Documents (No links in/out)

**Problem**: Orphaned docs are invisible, won't be found or maintained.

**How to Find**:
- Use Obsidian Graph View ‚Üí Look for isolated nodes
- Or use "Unlinked mentions" to find documents with no backlinks

**Fix Priority**: ‚≠ê LOW

**Action Steps**:

**For each orphan**:
1. **Link from README** - Add to relevant section README
2. **Cross-link** - Link to/from related docs
3. **Archive** - If truly obsolete, move to Archive/
4. **Delete** - If redundant or worthless (rare)

**Example**:
```markdown
Found orphan: Technical/Database/Query-Optimization.md

Fix:
1. Add to Technical/Database/README.md under "Performance" section
2. Link from Technical/Architecture/Database-Design.md
3. Link to related Technical/API/ docs that use these patterns
```

**Time Estimate**: 2-3 hours

## Best Practices for Future Development

### When Creating New Docs

**DO**:
- ‚úÖ Use [Templates/](/docs/engineering/ai-crews/readme) for consistency
- ‚úÖ Add frontmatter with type, status, owner, tags
- ‚úÖ Link from relevant README
- ‚úÖ Cross-link to related docs
- ‚úÖ Keep initial doc concise (aim for <500 lines)
- ‚úÖ Use subdirectories for complex features

**DON'T**:
- ‚ùå Create 1,000+ line monolithic docs
- ‚ùå Skip frontmatter "because it's quick"
- ‚ùå Create orphaned docs (link from somewhere)
- ‚ùå Duplicate existing content (search first)
- ‚ùå Leave TODO markers without timeline
- ‚ùå Avoid ownership (always assign owner)

---

### When Modifying Existing Docs

**DO**:
- ‚úÖ Update `last_reviewed` date in frontmatter
- ‚úÖ Remove completed TODOs immediately
- ‚úÖ Split if doc grows >1,000 lines
- ‚úÖ Add cross-links to new related docs
- ‚úÖ Archive if marking deprecated
- ‚úÖ Update README if structure changes

**DON'T**:
- ‚ùå Let TODOs accumulate
- ‚ùå Keep deprecated content in active sections
- ‚ùå Break links when renaming (use Obsidian rename)
- ‚ùå Remove content without archiving
- ‚ùå Forget to update related docs
