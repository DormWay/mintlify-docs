# Tag Taxonomy

Standardized tag hierarchy for the DormWay Obsidian vault. Use these tags consistently in frontmatter to enable powerful search and filtering.

---

## Domain Tags

Tags identifying the primary product area or service.

| Tag | Description | Use For |
|-----|-------------|---------|
| `#lockedin` | LockedIn student app (Next.js) | LockedIn features, pages, components |
| `#ios` | iOS native app (Swift/SwiftUI) | Mobile-specific implementations |
| `#backend` | Backend services | API Router, Engine, services |
| `#admin` | Admin dashboard | Admin features, tools |
| `#crews` | AI Crews service | CrewAI agents, orchestration |
| `#zuplo` | API Gateway | Auth, routing, rate limiting |
| `#canvas` | Canvas LMS integration | Sync, assignments, grades |
| `#temporal` | Workflow engine | Workflows, activities, signals |
| `#ably` | Real-time messaging | Channels, presence, events |

### Examples
```yaml
tags:
  - '#lockedin'
  - '#canvas'
```

## Usage Guidelines

### Frontmatter Format
```yaml
---
last_updated: 2025-12-11
owner: Platform
status: current
tags:
  - '#lockedin'
  - '#technical'
  - '#dashboard'
---
```

### Tag Selection Rules

1. **Always include at least one domain tag** - What area does this doc cover?
2. **Include one type tag** - What kind of document is it?
3. **Add component tags as relevant** - What specific components?
4. **Use status tags sparingly** - Prefer `status:` frontmatter field

### Combining Tags

Good combinations:
- `#lockedin #technical #dashboard` - LockedIn dashboard implementation
- `#backend #api #canvas` - Canvas API integration backend
- `#ios #architecture #widgets` - iOS widget architecture
- `#temporal #technical #engine` - Temporal workflow implementation

Avoid:
- Too many tags (max 5-6)
- Redundant tags (`#technical #implementation`)
- Overly specific tags (create component tags instead)

## Adding New Tags

Before creating a new tag:

1. Check if an existing tag covers the concept
2. Ensure it will be used by 3+ documents
3. Add it to this taxonomy with description
4. Update the search index

**Request new tags** by updating this document via PR.
