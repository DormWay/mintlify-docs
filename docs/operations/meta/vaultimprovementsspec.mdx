# Vault Improvements Specification

Comprehensive guide for maintaining and improving the DormWay Obsidian vault.

## Recommended Plugins

### Core Plugins (Required)

| Plugin | Purpose | Configuration |
|--------|---------|---------------|
| **Dataview** | Dynamic queries and tables | Enable JavaScript queries |
| **Templater** | Advanced templates | Set templates folder to `Templates/` |
| **Tag Wrangler** | Tag management and renaming | Enable tag rename propagation |

### Productivity Plugins (Recommended)

| Plugin | Purpose | Configuration |
|--------|---------|---------------|
| **Backlinks** | Built-in, shows incoming links | Enable in core plugins |
| **Outgoing links** | Shows document references | Enable in core plugins |
| **Graph view** | Visual relationship mapping | Enable in core plugins |
| **Quick Switcher** | Fast file navigation | Enable Cmd/Ctrl+O |
| **Search** | Full-text search | Enable in core plugins |

### Optional Plugins

| Plugin | Purpose | When to Use |
|--------|---------|-------------|
| **Calendar** | Daily notes calendar | If using Daily Notes |
| **Kanban** | Task boards | For project tracking |
| **Excalidraw** | Diagrams | For visual documentation |

## Plugin Installation

### Installing Community Plugins

1. Open Obsidian Settings (`Cmd+,`)
2. Navigate to Community Plugins
3. Click "Browse" to find plugins
4. Install and enable required plugins

### Dataview Setup

```
Settings > Community Plugins > Dataview
- Enable JavaScript Queries: ON
- Inline Queries: ON
- Inline JS Queries: ON
```

### Templater Setup

```
Settings > Community Plugins > Templater
- Template folder location: Templates/
- Trigger Templater on new file creation: ON
```

## Frontmatter Template

Every document should include YAML frontmatter:

```yaml
---
last_updated: 2025-12-11
owner: Platform | Design | Engineering | Documentation
status: current | in-progress | deprecated | archived
tags:
  - '#domain-tag'
  - '#type-tag'
  - '#component-tag'
aliases:
  - Alternative Name
  - Abbreviation
---
```

### Field Descriptions

| Field | Required | Description | Example |
|-------|----------|-------------|---------|
| `last_updated` | Yes | ISO date of last update | `2025-12-11` |
| `owner` | Yes | Team or person responsible | `Platform` |
| `status` | Yes | Document lifecycle status | `current` |
| `tags` | Yes | Categorization tags | `['#technical', '#api']` |
| `aliases` | No | Alternative search names | `['BFF', 'Backend for Frontend']` |

### Status Values

| Status | Description | Search Visibility |
|--------|-------------|-------------------|
| `current` | Active, up-to-date | High |
| `in-progress` | Being written/updated | Medium |
| `deprecated` | Superseded, for reference | Low |
| `archived` | Historical only | Minimal |

## Alias Guidelines

Aliases improve search discoverability. Add aliases for:

### Abbreviations
```yaml
aliases:
  - BFF
  - Backend for Frontend
```

### Alternative Names
```yaml
aliases:
  - StudentWatcher
  - Digital Twin
  - Student Watcher
```

### Common Misspellings
```yaml
aliases:
  - Dayplan
  - Day Plan
  - DayPlan
```

### Linear Issue References
```yaml
aliases:
  - DORM-455
  - Course Context Creation
```

### Best Practices

1. **Include acronyms** - Always add both spelled out and abbreviated
2. **Add Linear issues** - Link tickets to documentation
3. **Consider typos** - Add common misspellings
4. **Use variations** - CamelCase, spaces, hyphens

## Dataview Query Patterns

### List Documents in Folder
```dataview
LIST
FROM "Technical/API"
WHERE file.name != "README"
SORT file.name ASC
```

### Table with Metadata
```dataview
TABLE status, last_updated, owner
FROM "Architecture"
WHERE status = "current"
SORT last_updated DESC
```

### Filter by Tag
```dataview
TABLE tags, status
FROM ""
WHERE contains(tags, "#lockedin")
LIMIT 20
```

### Recently Updated
```dataview
TABLE last_updated, owner
FROM ""
WHERE last_updated
SORT last_updated DESC
LIMIT 10
```

### Find Deprecated Docs
```dataview
LIST
FROM ""
WHERE status = "deprecated"
```

### Count Documents by Status
```dataview
TABLE length(rows) as Count
FROM ""
GROUP BY status
```

### Documents Missing Frontmatter
```dataview
LIST
FROM ""
WHERE !last_updated
LIMIT 20
```

## Search Tips

### Using the Obsidian Search

| Search | Description |
|--------|-------------|
| `tag:#lockedin` | Find by tag |
| `path:Architecture` | Search in folder |
| `file:INDEX` | Find by filename |
| `line:dashboard` | Search content |
| `section:Overview` | Search in headings |

### Using the Custom Toolchain

The workspace provides BM25 search via `scripts/obsidian-tools/`:

```bash
# Search all docs
npm run obsidian:search -- "dashboard architecture"

# Search with limit
npm run obsidian:search -- "canvas sync" --limit 10

# Get document content
node retrieve.js "Architecture/README.md"
```

## Folder Organization

### Standard Folder Structure

```
DormWay/
├── Architecture/        # System design decisions
│   └── INDEX.md        # MOC for architecture
├── Design/             # UX/UI design docs
│   └── INDEX.md        # MOC for design
├── Technical/          # Implementation details
│   └── INDEX.md        # MOC for technical
├── Plans/              # Implementation plans
│   └── INDEX.md        # MOC for plans
├── Features/           # Feature specifications
├── Engineering/        # Engineering practices
├── Meta/               # Vault meta-documentation
├── Templates/          # Document templates
├── Archive/            # Historical content
└── Index.md           # Main vault index
```

### Creating New Folders

1. Create folder with appropriate name
2. Add `INDEX.md` or `README.md` as entry point
3. Update parent folder's index
4. Add to root `Index.md` if major section

## Troubleshooting

### Dataview Queries Not Rendering
1. Check Dataview plugin is installed and enabled
2. Verify query syntax is correct
3. Check frontmatter YAML formatting

### Search Not Finding Documents
1. Run search index rebuild: `npm run obsidian:build-index`
2. Check document has frontmatter
3. Verify aliases are set

### Broken Links
1. Use Obsidian's "Show broken links" feature
2. Run: `npm run obsidian:doctor`
3. Update wiki links after moves
