## Executive Summary

The Academic page is **DormWay's deepest academic intelligence layer** - where students go beyond the "desk" (home dashboard) to manage their full courseload, assignments, grades, and study planning. This document outlines a student-first evolution based on:

- **Current Implementation Analysis**: Academic Workstation V2 with time-block planning
- **Genuine Student Use Cases**: Research-backed academic management patterns
- **Progressive Disclosure Principles**: Information when you need it, nothing more
- **Deep Linking Strategy**: Direct access to assignments in Canvas/Blackboard/Moodle
- **Multi-Device Reality**: Consistent experience across iOS, web, and mobile

**Core Philosophy**: *"One stop shop for academics, without overwhelming cognitive load"*

## Student-First Use Cases

### Primary Use Cases (80% of Activity)

**1. "What's due soon?"** (Daily check-in)
- Student opens academic page
- Sees urgent items first (Overdue â†’ Today â†’ This Week)
- Clicks item â†’ Inspector shows details
- Takes action: Start work, Plan time, or Mark complete
- **Goal**: Zero cognitive load to see priorities

**2. "When should I work on this?"** (Time management)
- Student sees assignment with 5h estimate
- Checks Capacity Bar: 3h already scheduled, need 2h more
- Clicks "Plan" â†’ Planner Drawer opens
- Selects: Tomorrow 2-4pm, creates time block
- Capacity Bar updates, anxiety reduced
- **Goal**: Proactive planning beats last-minute panic

**3. "How am I doing in this class?"** (Course health check)
- Student filters by course (e.g., CS 101)
- Sees pending assignments, recent grades, next deadline
- Clicks course card â†’ Expanded view with full details
- Reviews syllabus policies, late work rules
- **Goal**: Course-level awareness without digging through LMS

**4. "What grade do I need on the final?"** (Grade planning)
- Student navigates to Gradebook tab
- Sees current course average: 78%
- Uses what-if calculator: "If I get 90% on final (30% weight)..."
- Calculator shows: "Final grade: 81.6% (B-)"
- **Goal**: Remove grade anxiety with concrete data

**5. "I need to actually do this assignment"** (Deep work mode)
- Student finds assignment in list
- Clicks "View in Canvas" button â†’ Opens Canvas assignment page
- Sees full assignment details, rubric, submission box
- Submits work directly in LMS
- Returns to DormWay â†’ Marks as submitted
- **Goal**: Seamless transition to work environment

### Secondary Use Cases (20% of Activity)

**6. "What's my week look like?"** (Weekly planning)
- Student navigates to Timeline tab
- Sees visual timeline of all deadlines next 2 weeks
- Spots heavy days: 3 assignments due Tuesday
- Plans accordingly, moves lighter work to Monday
- **Goal**: Avoid surprise crunch days

**7. "What are the course policies?"** (Policy lookup)
- Student opens assignment inspector
- Sees late policy: "-10% per day, 3-day max"
- Checks slack days: 2 remaining
- Decides: Worth using slack day for quality work
- **Goal**: Policy-aware decision making

**8. "I finished everything, now what?"** (Proactive mode)
- Student sees empty "Due Soon" section
- Checks "Next Week" section: 4 assignments
- Plans ahead: Schedules study sessions for next week's work
- **Goal**: Reward proactive students, reduce last-minute stress

## Progressive Disclosure Strategy

### Principle: Information Hierarchy

**Level 1: Glance (Home Dashboard)**
- Show: Count of urgent assignments (e.g., "3 due today")
- Action: Click widget â†’ Navigate to Academic page

**Level 2: Scan (Academic Page - Default View)**
- Show: Grouped assignments (Overdue â†’ Today â†’ This Week â†’ Next Week)
- Each row: Course code, Title, Due date, Quick actions
- Capacity Bar: Required vs. Scheduled hours
- Action: Click row â†’ Inspector opens (Level 3)

**Level 3: Inspect (Assignment Inspector)**
- Show: Full details (description, points, weight, links, policies)
- Actions: Plan time, Start work, View in LMS, Mark complete
- Action: Click "View in Canvas" â†’ Opens LMS (Level 4)

**Level 4: Deep Work (LMS)**
- Student is now in Canvas/Blackboard/Moodle
- Sees full assignment, rubric, submission interface
- Submits work directly in LMS
- Returns to DormWay to update status

**Level 5: Analytics (Gradebook/Timeline)**
- Student wants deeper insights
- Navigates to Gradebook or Timeline tabs
- Sees trends, projections, timeline visualizations
- Action: Plan semester strategy

## Deep Linking Strategy

### Current State
- âœ… DormWay internal deep links work (`?item=<deliverableId>`)
- âŒ No links to actual LMS assignments
- âŒ Students must manually find assignment in Canvas/Blackboard

### Proposed Solution: LMS Deep Links

**Data Model Addition** (Backend)
```typescript
// Add to deliverable schema
interface Deliverable {
  // ... existing fields ...
  lmsUrl?: string;           // Full URL to assignment in LMS
  lmsType?: 'canvas' | 'blackboard' | 'moodle' | 'ics';
}
```

**Canvas Deep Link Pattern**
```
https://<institution>.instructure.com/courses/<courseId>/assignments/<assignmentId>

Example:
https://umich.instructure.com/courses/12345/assignments/67890
```

**Backend Implementation** (Engine/Canvas Activities)
```typescript
// When fetching Canvas assignments, include html_url
const assignment = await canvasAPI.getAssignment(courseId, assignmentId);

await db.query(`
  UPDATE service_data
  SET data = jsonb_set(data, '{lmsUrl}', $1, true),
      data = jsonb_set(data, '{lmsType}', '"canvas"', true)
  WHERE id = $2
`, [assignment.html_url, deliverableId]);
```

**Frontend Implementation** (Academic Inspector)
```tsx
// In AcademicInspector.tsx
{deliverable.lmsUrl && (
  <a
    href={deliverable.lmsUrl}
    target="_blank"
    rel="noopener noreferrer"
    className="btn-primary"
    onClick={() => track('assignment_lms_link_clicked', {
      assignmentId: deliverable.id,
      lmsType: deliverable.lmsType
    })}
  >
    <ExternalLink className="w-4 h-4" />
    View in {deliverable.lmsType === 'canvas' ? 'Canvas' : 'LMS'}
  </a>
)}
```

**Benefits**:
- âœ… One-click access to actual assignment
- âœ… Students can submit work directly in LMS
- âœ… Reduces context switching friction
- âœ… Maintains LMS as source of truth for submissions

**Fallback for ICS-based assignments**:
- If `lmsType === 'ics'` (no direct LMS link)
- Show message: "This assignment was imported from your calendar. Visit your course's LMS page to submit work."
- Provide course homepage link if available

## Smart Prioritization (V3 Feature)

### Problem
Students see all assignments equally. No smart "Focus on this first" guidance.

### Solution: Intelligent Priority Scoring

**Priority Algorithm**:
```typescript
function calculatePriority(deliverable: Deliverable): number {
  let score = 0;

  // Time urgency (0-40 points)
  const hoursUntilDue = (new Date(deliverable.dueAt).getTime() - Date.now()) / (1000 * 60 * 60);
  if (hoursUntilDue < 0) score += 40;        // Overdue
  else if (hoursUntilDue < 12) score += 35;  // Due today
  else if (hoursUntilDue < 48) score += 30;  // Due tomorrow
  else if (hoursUntilDue < 168) score += 20; // Due this week

  // Weight/importance (0-25 points)
  if (deliverable.weightPct >= 30) score += 25;  // Major assignment
  else if (deliverable.weightPct >= 20) score += 20;
  else if (deliverable.weightPct >= 10) score += 15;
  else score += 10;

  // Estimated effort (0-20 points)
  if (deliverable.estHours >= 10) score += 20;  // Long assignment needs early start
  else if (deliverable.estHours >= 5) score += 15;
  else if (deliverable.estHours >= 2) score += 10;
  else score += 5;

  // Time scheduled (0-15 points penalty)
  const hoursScheduled = getScheduledHours(deliverable.id);
  if (hoursScheduled === 0) score += 15;        // Nothing scheduled, high priority
  else if (hoursScheduled < deliverable.estHours) score += 10;
  else score += 0;  // Fully scheduled, lower priority

  return score;
}
```

**Conflict Detection Logic**:
```typescript
interface WorkloadDay {
  date: string;
  assignments: Deliverable[];
  totalHours: number;
  totalWeight: number;  // Sum of assignment weights
  risk: 'low' | 'medium' | 'high' | 'critical';
}

function analyzeWeek(deliverables: Deliverable[]): WorkloadDay[] {
  const days = groupByDay(deliverables);

  return days.map(day => {
    const totalHours = day.assignments.reduce((sum, a) => sum + (a.estHours || 0), 0);
    const totalWeight = day.assignments.reduce((sum, a) => sum + (a.weightPct || 0), 0);

    let risk: WorkloadDay['risk'] = 'low';
    if (totalHours >= 12 || totalWeight >= 60) risk = 'critical';
    else if (totalHours >= 8 || totalWeight >= 40) risk = 'high';
    else if (totalHours >= 5 || totalWeight >= 25) risk = 'medium';

    return { ...day, totalHours, totalWeight, risk };
  });
}
```
