## Overview

The Academic Workstation V2 transforms the Academic page from a full-screen modal experience into a **multitaskable 3-column workstation** with intelligent filtering, deep-linking, and time-block planning capabilities. This redesign prioritizes battery life, performance, and cognitive ease for Gen-Z students.

### Key Principle
**"Student OS" mentality** â€” No modal lockouts. Students should be able to browse, plan, and reference simultaneously without context switching.

---

## Problem Statement

### What Was Broken in V1

1. **Blocking Modal Inspector**
   - Full-screen backdrop locked entire UI
   - Students couldn't browse other assignments while viewing details
   - Forced context switching (close â†’ browse â†’ reopen)

2. **Confusing Filter Logic**
   - Filters used AND-only logic across all categories
   - Selecting "Today" + "This Week" showed nothing (impossible combo)
   - Students expected OR-within category, got frustration

3. **No Link Sharing**
   - No URL state for specific assignments
   - Couldn't share "check out this assignment" links
   - Browser back/forward didn't work as expected

4. **Performance Issues**
   - `transition-all` caused layout thrashing
   - Backdrop blur drained 30% extra battery
   - Animations lagged on older devices (2019 MacBook Air)

5. **No Planning Tools**
   - No way to schedule study time
   - No visibility into planned vs. required hours
   - Guesswork on time allocation

### 2. Intelligent Filtering (OR-within / AND-across)

**Logic:**
```typescript
// OR within a category
timeFilters.some(f => matches)  // "Today" OR "This Week"

// AND across categories
matchesTime && matchesType && matchesSource && matchesCourse
```

**Example:**
- Select: `[Today]` `[This Week]` â†’ Shows assignments due today **OR** this week
- Add: `[Exam]` â†’ Shows exams due today OR this week
- Add: `[MATH 101]` â†’ Shows MATH 101 exams due today OR this week

**Benefits:**
- Intuitive chip-based UI
- Zero impossible combinations
- Students get what they expect

- **Framework:** Next.js 14 (App Router)
- **Language:** TypeScript (strict mode)

## API Integration

### Time Blocks API

**Endpoints:**
```typescript
POST   /api/proxy/time-blocks      // Create manual study session
GET    /api/proxy/time-blocks?week=YYYY-WW  // Get week's blocks
DELETE /api/proxy/time-blocks/:id  // Soft delete block
```

**Request/Response:**
```typescript
// Create
POST /api/proxy/time-blocks
{
  "taskId": "deliverable-uuid",
  "startISO": "2025-10-26T14:00:00Z",
  "minutes": 60
}

// Response
{
  "id": "block-uuid",
  "start_time": "2025-10-26T14:00:00Z",
  "end_time": "2025-10-26T15:00:00Z",
  "task_id": "deliverable-uuid",
  "duration_minutes": 60
}

// Get Week
GET /api/proxy/time-blocks?week=2025-42
{
  "week": "2025-42",
  "blocks": [/* TimeBlock[] */],
  "totalHours": 7.5
}
```

### Database Schema

**Table:** `time_blocks`

```sql
CREATE TABLE time_blocks (
  id uuid PRIMARY KEY,
  user_id uuid NOT NULL,
  start_time timestamptz NOT NULL,
  end_time timestamptz NOT NULL,
  title text NOT NULL,
  source_type time_block_source_type NOT NULL,  -- 'manual' for student-created
  source_id text NOT NULL,  -- deliverable ID
  event_type time_block_type,
  lifecycle_status time_block_lifecycle_status DEFAULT 'active',
  -- ... other fields
);
```

**Key Points:**
- `source_type='manual'` for student-created blocks
- `lifecycle_status='active'` (soft delete pattern)
- `source_id` links to deliverable/assignment
