# Customer.io Campaign Map - 5 User Journeys

This document maps out all Customer.io campaigns, email series, and triggered events for the 5 core user journeys launching with DormWay.

## Overview

Each journey has specific triggered events that should fire Customer.io campaigns. These include:
- **Push notifications** (via Customer.io Push API)
- **Email campaigns** (onboarding, re-engagement, feature education)
- **In-app messages** (tooltips, success confirmations)
- **Analytics events** (for tracking and triggering)

## Journey 1: First-Time Onboarding

### Triggered Events

1. **user_created**
   - Welcome email series (3 emails over 7 days)
   - Push notification permission prompt (Day 2)
   
2. **onboarding_started**
   - Track progress through each step
   
3. **campus_selected**
   - Campus-specific welcome content
   
4. **major_selected**
   - Major-specific tips and resources
   
5. **permissions_granted**
   - Thank you + quick tips on what's now enabled
   
6. **onboarding_completed**
   - Celebration push + next steps email

1. **syllabus_upload_started**
   - Track upload initiation
   
2. **syllabus_processing_started**
   - Show processing status
   
3. **syllabus_processed_success**

4. **syllabus_processing_failed**

   - Support email with troubleshooting

## Journey 5: Daily Return User

### Triggered Events

1. **app_opened_daily**
   - Track DAU
   
2. **morning_context_loaded**
   - Track context system
   
3. **notification_preference_changed**
   - Update delivery preferences

   - Track engagement

   - Feature adoption
   
6. **study_session_completed**
   - Engagement tracking

### Recurring Campaigns

**Morning Briefing** (Daily, 8 AM user time)
- Push: Contextual summary based on schedule
- Variations: Class day, study day, deadline day, weekend

**Midday Check-in** (MWF, 12 PM user time)
- Push: Progress check, motivation, or deadline reminder
- Smart timing based on calendar gaps

**Evening Review** (Daily, configurable time)
- Push: Tomorrow prep, completion celebration, or wellness check
- Adaptive based on today's activity

**Weekly Digest Email** (Sunday evening)
- Subject: "Your week in review + what's ahead"
- Content: Accomplishments, upcoming deadlines, study suggestions
- CTA: Plan your week

### Re-engagement Campaigns

**Dormant User Win-back** (No app open in 3 days)
- Day 3: Gentle reminder push
- Day 5: "Missing something?" email
- Day 7: Important deadline reminder (if applicable)
- Day 14: "We're here when you need us" final email

**Feature Discovery** (For users not using key features)
- BrainGains prompt (if no questions in 7 days)
- Calendar reminder (if not connected after 14 days)
- Notification opt-in (if declined but engaging otherwise)

## Technical Implementation Notes

### Customer.io Setup Required

1. **User Attributes**
   - user_id
   - email
   - first_name
   - campus
   - major
   - timezone
   - notification_preferences
   - feature_flags
   - engagement_scores

2. **Event Tracking**
   - All events listed above must be fired from iOS app
   - Include relevant metadata (course names, counts, etc.)
   - Use consistent naming convention

3. **Segments**
   - New users (< 7 days)
   - Engaged users (daily active)
   - At-risk users (3-6 days inactive)
   - Churned users (7+ days inactive)
   - Feature-specific segments (has_syllabus, has_calendar, etc.)

4. **Campaign Timing**
   - Respect user timezone
   - Respect quiet hours preference
   - Use intelligent delivery (Customer.io feature)
   - Implement frequency caps

5. **Templates Needed**
   - Email templates for each campaign
   - Push notification templates with personalization
   - A/B test variations for key messages

### Integration Points

1. **iOS App**
   - Fire events via Customer.io SDK
   - Update user attributes on changes
   - Handle deep links from campaigns

2. **Backend**
   - Sync user preferences to Customer.io
   - Trigger transactional emails via API
   - Update engagement scores

3. **Analytics**
   - Track campaign performance
   - Monitor engagement rates
   - A/B test results
