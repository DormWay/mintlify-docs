# Hero Section Visual Analysis - Complete Breakdown

## Overview
This document explains **every single factor** that impacts the visual appearance of the landing page hero section, from the outermost container to the innermost demo iframe.

---

## Container Hierarchy (Outer to Inner)

### 1. `<section>` - Hero Section Wrapper
**Location:** Line 1221
**Classes:** `relative pt-[var(--ds-spacing-lg)] pb-16 3xl:pt-16 3xl:pb-24`

**Impact:**
- **Top Padding:** `pt-[var(--ds-spacing-lg)]` = **24px** (1.5rem)
  - This creates space between the navbar and hero content
  - **ISSUE:** This is LESS than navbar's top padding, which uses the same value but also has bottom padding
- **Bottom Padding:** `pb-16` = **64px** (4rem) on all sizes
  - `3xl:pb-24` = **96px** (6rem) on screens ≥1920px
- **Position:** `relative` - allows absolute positioning of children

**Visual Effect:** Controls vertical spacing above and below the entire hero section.

---

### 2. `<div>` - Main Container (CRITICAL CONSTRAINT)
**Location:** Line 1222
**Classes:** `mx-auto px-3 sm:px-4 md:px-[var(--ds-spacing-lg)] lg:px-[var(--ds-spacing-xl)] max-w-[1200px] xl:max-w-[1400px] 3xl:max-w-[1720px] 4xl:max-w-[1960px]`

**Impact:**
- **Horizontal Centering:** `mx-auto` - centers the container
- **Horizontal Padding (LEFT/RIGHT):**
  - Mobile: `px-3` = **12px**
  - Small (≥640px): `px-4` = **16px**
  - Medium (≥768px): `px-[var(--ds-spacing-lg)]` = **24px**
  - Large (≥1024px): `px-[var(--ds-spacing-xl)]` = **32px**
  
- **Max-Width Constraints (CRITICAL - THIS IS LIMITING THE DEMO):**
  - Default: `max-w-[1200px]` = **1200px maximum width**
  - XL (≥1280px): `max-w-[1400px]` = **1400px maximum width**
  - 3XL (≥1920px): `max-w-[1720px]` = **1720px maximum width**
  - 4XL (≥2560px): `max-w-[1960px]` = **1960px maximum width**

**Visual Effect:** 
- **THIS IS THE PRIMARY CONSTRAINT** preventing the demo from being 2/3 of the screen width
- The container is capped at 1200px-1960px, so even if the screen is 3000px wide, the content only uses 1960px max
- The padding further reduces available space (subtract 24-32px on each side)

**Available Width Calculation:**
- On a 1920px screen with 3xl breakpoint:
  - Container max-width: 1720px
  - Padding left: 32px
  - Padding right: 32px
  - **Available content width: 1720 - 32 - 32 = 1656px**

---

### 3. `<div>` - Grid Container
**Location:** Line 1223
**Classes:** `grid items-center justify-items-center gap-8 lg:gap-12 grid-cols-1 xl:grid-cols-[minmax(360px,1fr)_minmax(720px,2fr)]`

**Impact:**
- **Grid Layout:**
  - Mobile/Tablet: `grid-cols-1` = **single column** (stacks vertically)
  - XL (≥1280px): `xl:grid-cols-[minmax(360px,1fr)_minmax(720px,2fr)]` = **two columns with 1:2 ratio**
  
- **Column Sizing:**
  - **Left Column:** `minmax(360px,1fr)`
    - Minimum width: **360px**
    - Maximum: **1fr** (1/3 of available grid space)
  - **Right Column:** `minmax(720px,2fr)`
    - Minimum width: **720px**
    - Maximum: **2fr** (2/3 of available grid space)
  
- **Gap Between Columns:**
  - Default: `gap-8` = **32px**
  - Large (≥1024px): `lg:gap-12` = **48px**

- **Alignment:**
  - `items-center` - vertically centers grid items
  - `justify-items-center` - horizontally centers grid items

**Visual Effect:**
- Creates the 1:2 ratio between left text and right demo
- **BUT** the available space is already constrained by the parent container's max-width
- The gap (32-48px) further reduces space available to columns

**Actual Column Width Calculation (on 1920px screen):**
- Available grid width: 1656px (from container calculation above)
- Gap: 48px
- **Total column space: 1656 - 48 = 1608px**
- Left column: 1608px ÷ 3 = **536px**
- Right column: 1608px × 2/3 = **1072px**

**ISSUE:** The demo column is only getting ~1072px on a 1920px screen, not 2/3 of the full screen width (which would be ~1280px).

---

### 4. Left Column - Text Content
**Location:** Line 1225
**Classes:** `hero text-center space-y-6 min-w-0 max-w-[68ch] 3xl:max-w-[70ch]`

**Impact:**
- **Max Width:** `max-w-[68ch]` = **68 character widths** (roughly 544-680px depending on font)
  - `3xl:max-w-[70ch]` = **70 character widths** on ≥1920px screens
- **Min Width:** `min-w-0` - allows shrinking below content size
- **Spacing:** `space-y-6` = **24px** vertical gap between children
- **Alignment:** `text-center` - centers all text

**Visual Effect:** 
- Prevents text from being too wide (readability)
- Centers content within its column
- The `max-w-[68ch]` constraint means even if the column is 536px wide, the text only uses ~544-680px max

---

### 5. Right Column - Demo Container
**Location:** Line 1271
**Classes:** `relative w-full min-w-0`

**Impact:**
- **Width:** `w-full` = **100% of grid column width** (so ~1072px on 1920px screen)
- **Min Width:** `min-w-0` - allows shrinking
- **Position:** `relative` - allows absolute positioning of children

**Visual Effect:** 
- Takes up the full width of its grid column
- **BUT** the grid column itself is constrained by the parent container's max-width

---

### 6. Demo Frame Wrapper
**Location:** Line 1272
**Classes:** `rounded-2xl border border-[var(--ds-border-subtle)] bg-[var(--ds-surface-1)] p-1 shadow-[var(--ds-shadow-lg)] w-full`

**Impact:**
- **Width:** `w-full` = **100% of parent** (so ~1072px)
- **Padding:** `p-1` = **4px** on all sides
- **Border:** `border` = **1px** border (subtracts from available space)
- **Border Radius:** `rounded-2xl` = **16px** rounded corners

**Visual Effect:**
- Creates the laptop frame appearance
- The `p-1` padding and border reduce the inner space by ~10px total

**Available Space for Demo:** ~1072px - 4px (left padding) - 4px (right padding) - 1px (left border) - 1px (right border) = **~1062px**

---

### 7. Demo Iframe Container (THE ACTUAL DEMO)
**Location:** Line 1273
**Classes:** `rounded-xl overflow-hidden bg-black aspect-video md:aspect-[7/4] xl:aspect-video relative w-full min-h-[400px]`

**Impact:**
- **Width:** `w-full` = **100% of parent** (so ~1062px)
- **Aspect Ratio:**
  - Mobile: `aspect-video` = **16:9** (1.777:1)
  - Medium (≥768px): `md:aspect-[7/4]` = **7:4** (1.75:1) - slightly taller
  - XL (≥1280px): `xl:aspect-video` = **16:9** (1.777:1)
- **Min Height:** `min-h-[400px]` = **400px minimum height**
- **Background:** `bg-black` - black background

**Visual Effect:**
- The demo's width is **~1062px** on a 1920px screen
- Height is determined by aspect ratio: 1062px ÷ 1.777 = **~598px tall**
- **ISSUE:** This is NOT 2/3 of the screen width (which would be ~1280px on a 1920px screen)

## The Problem: Why Demo Isn't 2/3 of Screen Width

### Current Flow:
1. **Screen Width:** 1920px
2. **Container Max-Width:** 1720px (constraint #1)
3. **Container Padding:** 32px left + 32px right = 64px total
4. **Available Content Width:** 1720 - 64 = **1656px**
5. **Grid Gap:** 48px
6. **Available Column Space:** 1656 - 48 = **1608px**
7. **Right Column (2fr):** 1608px × 2/3 = **1072px**
8. **Frame Padding/Border:** -10px
9. **Final Demo Width:** **~1062px**

### What You Want:
1. **Screen Width:** 1920px
2. **Demo Should Be:** 1920px × 2/3 = **1280px**

### The Gap:
- **Current:** ~1062px
- **Desired:** ~1280px
- **Difference:** ~218px (about 20% smaller than intended)

---

## All Constraints Affecting Demo Size

### Primary Constraints (In Order of Impact):

1. **Container Max-Width** (Line 1222)
   - `max-w-[1200px]` to `max-w-[1960px]`
   - **Impact:** Prevents content from using full screen width
   - **Solution:** Remove or significantly increase these values

2. **Container Horizontal Padding** (Line 1222)
   - `px-3` to `px-[var(--ds-spacing-xl)]` = 12px to 32px per side
   - **Impact:** Reduces available width by 24-64px total
   - **Solution:** Match navbar padding exactly, or reduce if needed

3. **Grid Gap** (Line 1223)
   - `gap-8` to `gap-12` = 32px to 48px
   - **Impact:** Reduces available column space
   - **Solution:** Reduce gap if demo needs more space

4. **Frame Padding/Border** (Line 1272)
   - `p-1` + `border` = ~10px total reduction
   - **Impact:** Small but adds up
   - **Solution:** Minimal impact, can keep

5. **Aspect Ratio** (Line 1273)
   - `aspect-video` = 16:9
   - **Impact:** Determines height based on width
   - **Solution:** Keep for proper proportions

## Text Sizing Hierarchy

### Main Headline (h1)
**Location:** Line 1229-1230
**Classes:** `text-[clamp(2.5rem,6vw,4rem)] sm:text-[clamp(3rem,7vw,4.5rem)] md:text-[clamp(3.5rem,8vw,5rem)] lg:text-[clamp(4rem,9vw,5.5rem)] 3xl:text-[clamp(4.5rem,10vw,6rem)] 4xl:text-[clamp(5rem,11vw,7rem)]`

**Impact:**
- Uses `clamp()` for fluid sizing
- **Mobile:** 2.5rem (40px) to 4rem (64px)
- **Small:** 3rem (48px) to 4.5rem (72px)
- **Medium:** 3.5rem (56px) to 5rem (80px)
- **Large:** 4rem (64px) to 5.5rem (88px)
- **3XL:** 4.5rem (72px) to 6rem (96px)
- **4XL:** 5rem (80px) to 7rem (112px)

**Visual Effect:** 
- Scales with viewport width (vw units)
- Maintains minimum and maximum sizes
- **ISSUE:** On very narrow screens, might still be too small relative to other text

### Sub-headline (p)
**Location:** Line 1234
**Classes:** `text-lg sm:text-xl md:text-2xl`

**Impact:**
- **Mobile:** `text-lg` = 18px

**Visual Effect:** Should always be smaller than headline, but might compete on narrow screens

### Problem Statement (p)

**Impact:**

## Recommendations to Fix Demo Size

### Option 1: Remove Container Max-Width (Recommended)
**Change Line 1222:**
```tsx
// FROM:
<div className="mx-auto px-3 sm:px-4 md:px-[var(--ds-spacing-lg)] lg:px-[var(--ds-spacing-xl)] max-w-[1200px] xl:max-w-[1400px] 3xl:max-w-[1720px] 4xl:max-w-[1960px]">

// TO:
<div className="mx-auto px-3 sm:px-4 md:px-[var(--ds-spacing-lg)] lg:px-[var(--ds-spacing-xl)]">
```

**Impact:** Content can now use full screen width minus padding

### Option 2: Increase Container Max-Width Significantly
**Change Line 1222:**
```tsx
// TO:
<div className="mx-auto px-3 sm:px-4 md:px-[var(--ds-spacing-lg)] lg:px-[var(--ds-spacing-xl)] max-w-[1600px] xl:max-w-[2000px] 3xl:max-w-[2400px] 4xl:max-w-[2800px]">
```

**Impact:** Allows more width while still maintaining some constraint

### Option 3: Make Grid Use Viewport Width
**Change Line 1223:**
```tsx
// TO:
<div className="grid items-center justify-items-center gap-8 lg:gap-12 grid-cols-1 xl:grid-cols-[minmax(360px,33.33vw)_minmax(720px,66.67vw)]">
```

**Impact:** Columns use viewport width directly, bypassing container constraints
**Note:** This might cause issues on very wide screens

### Option 4: Move Grid Outside Container
**Restructure:**
- Move the grid container outside the max-width container
- Apply padding directly to the grid
- Let grid columns use full viewport width

---

## Summary: Why It's Not Working

1. **Container max-width** is capping the entire hero at 1200-1960px
2. **Container padding** further reduces available space (24-64px total)
3. **Grid gap** reduces column space (32-48px)
4. **Frame padding/border** reduces demo space (~10px)
5. **Result:** Demo gets ~1062px on a 1920px screen instead of ~1280px

**The fix:** Remove or significantly increase the container max-width constraints, and ensure the grid can use the full viewport width (minus padding).
