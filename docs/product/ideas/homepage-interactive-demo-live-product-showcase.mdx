### Boot Loading Script
```typescript
// scripts/seed-demo-account.ts
import { db } from '@/lib/database'
import { DEMO_COURSES, DEMO_ASSIGNMENTS, DEMO_SYLLABI } from './demo-data'

async function seedDemoAccount() {
  const demoUserId = 'demo@dormway.app'

  // 1. Ensure demo user exists
  await db.users.upsert({
    where: { email: demoUserId },
    update: {},
    create: {
      email: demoUserId,
      displayName: 'Demo Student',
      campus_id: 'umich-ann-arbor',
      created_at: new Date()
    }
  })

  // 2. Clear existing demo data (reset to baseline)
  await db.courses.deleteMany({ where: { user_id: demoUserId } })
  await db.assignments.deleteMany({ where: { user_id: demoUserId } })
  await db.syllabi.deleteMany({ where: { user_id: demoUserId } })

  // 3. Insert courses
  for (const course of DEMO_COURSES) {
    await db.courses.create({
      data: {
        user_id: demoUserId,
        code: course.code,
        title: course.title,
        credits: course.credits,
        section: course.section,
        instructor: course.instructor,
        schedule: course.schedule,
        term: 'Fall 2024'
      }
    })
  }

  // 4. Insert assignments
  for (const assignment of DEMO_ASSIGNMENTS) {
    await db.assignments.create({
      data: {
        user_id: demoUserId,
        course_code: assignment.course,
        type: assignment.type,
        title: assignment.title,
        due_date: new Date(assignment.due),
        points: assignment.points,
        estimated_hours: assignment.estimated_hours
      }
    })
  }

  // 5. Insert syllabi metadata
  for (const syllabus of DEMO_SYLLABI) {
    await db.syllabi.create({
      data: {
        user_id: demoUserId,
        course_code: syllabus.course,
        filename: syllabus.filename,
        policies: syllabus.policies,
        uploaded_at: new Date()
      }
    })
  }

  // 6. Generate intelligence insights (via Engine workflow)
  await triggerIntelligenceGeneration(demoUserId)

  console.log('âœ… Demo account seeded successfully')
}

// Run weekly via cron or Temporal workflow
```

## iframe Communication (postMessage API)

**Parent â†’ iframe**: Theme switching
```typescript
// In HomepageDemo.tsx (parent)
const iframeRef = useRef<HTMLIFrameElement>(null)

function switchTheme(theme: 'light' | 'dark') {
  iframeRef.current?.contentWindow?.postMessage(
    { type: 'THEME_CHANGE', theme },
    window.location.origin
  )
}
```

**iframe â†’ Parent**: Event tracking
```typescript
// In demo mode layout (inside iframe)
useEffect(() => {
  if (isDemoMode) {
    window.parent.postMessage(
      { type: 'DEMO_EVENT', event: 'page_view', data: { route: '/dashboard/home' } },
      window.location.origin
    )
  }
}, [isDemoMode, route])
```

**Security**: Only accept messages from same origin
```typescript
window.addEventListener('message', (event) => {
  if (event.origin !== window.location.origin) return // Security check

  if (event.data.type === 'THEME_CHANGE') {
    setTheme(event.data.theme)
  }
})
```

## Security & Privacy

### Demo Account Protection
**Read-Only Mode**: Enforce at multiple layers
```typescript
// Middleware in api-router
if (req.user.email === 'demo@dormway.app' && req.method !== 'GET') {
  return res.status(403).json({ error: 'Demo account is read-only' })
}
```

**No PII (Personally Identifiable Information)**:
- âŒ No real student names (use "Demo Student")
- âŒ No real email addresses (only demo@dormway.app)
- âŒ No Riley's personal data (privacy violation)
- âœ… Synthetic U of M course data (public course catalog info)
- âœ… Generic instructor names ("Prof. Demo", "Prof. Sample")
- âœ… Realistic but fake assignment titles

**Rate Limiting**: Prevent abuse of demo account
```typescript
// Limit demo account to 100 requests/minute
const rateLimiter = rateLimit({
  windowMs: 60 * 1000, // 1 minute
  max: (req) => req.user.email === 'demo@dormway.app' ? 100 : 1000
})
```

### iframe Security
**Same-Origin Policy**: iframe loads from same domain
- âœ… iframe src: `https://dormway.app/dashboard/home?demo=true`
- âŒ Not: `https://demo.dormway.app` (separate subdomain)
- **Reason**: Easier postMessage communication, no CORS issues

**Content Security Policy (CSP)**:
```http
Content-Security-Policy: frame-ancestors 'self'
```
Prevents demo pages from being iframed by external sites

**X-Frame-Options**: Allow same-origin framing only
```http
X-Frame-Options: SAMEORIGIN
```

## Risk Mitigation

### Performance Risk
**Risk**: iframe loads slowly, impacting homepage performance
**Mitigation**:
- Lazy load iframe (only when user scrolls to demo section)
- Use `loading="lazy"` attribute on iframe
- Cache demo account data in Redis (no DB queries on every view)
- Add skeleton loader while iframe loads
- Test with slow 3G throttling

### Complexity Risk
**Risk**: 48 hours is still significant for a conversion experiment
**Mitigation**:
- Build Phase 1 only (12h), test with static demo data first
- If conversion lift > 10%, continue to Phases 2-4
- If lift < 5%, reassess approach (maybe video demo instead)
- Track analytics closely (time in demo, tab switches, etc.)

### User Confusion Risk
**Risk**: Users think demo is their actual account
**Mitigation**:
- Persistent "DEMO MODE" badge in top-right of iframe
- Different color accent (blue instead of purple?)
- Clear CTA: "Sign up to build YOUR plan" (not "Continue" or "Save")
- Show demo email (demo@dormway.app) in any visible UI

### iframe Compatibility Risk
**Risk**: iframes behave differently on mobile Safari, some browsers
**Mitigation**:
- Test on iOS Safari (strictest iframe policies)
- Test on Android Chrome, Firefox
- Fallback: If iframe fails to load, show screenshot + video demo
- Use `sandbox` attribute judiciously (allow-scripts, allow-same-origin)
  ```html
  <iframe sandbox="allow-scripts allow-same-origin" src="/dashboard/home?demo=true" />
  ```

### Data Staleness Risk
**Risk**: Demo data becomes outdated (past semester dates)
**Mitigation**:
- Automated weekly refresh via Temporal workflow
- Use rolling dates (e.g., "current week + 2", not hardcoded "Oct 18")
- Monitor demo account health with automated checks
- Alert if demo data is > 7 days old
