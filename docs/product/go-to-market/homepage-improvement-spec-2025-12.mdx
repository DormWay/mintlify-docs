# Homepage Improvement Spec - December 2025

> Simplify the homepage to 6 sections, remove the live demo iframe, and make PLG upload the centerpiece.

---

## Executive Summary

- **Decision**: Remove live demo iframe entirely - it's not common practice and adds complexity
- **Focus**: PLG upload flow (schedule, syllabus, GO) is the key acquisition path
- **Structure**: Simplify from 13 sections to 6 sections
- **Visual**: Replace iframe with "What You'll Get" demo data preview
- **Future**: Mobile app will be important; keep UI/UX flexible for evolution

---

## Why Live Demo Iframes Are Wrong

### Industry Practice

Live iframe demos are **not common practice**. Top PLG companies use static visuals:

| Company | Approach |
|---------|----------|
| Notion | Static screenshots + "Try Notion free" CTA |
| Linear | Animated product videos + sign up |
| Figma | Static visuals + "Get started for free" |
| Loom | Product video + CTA |
| Calendly | Static mockup + "Sign up free" |
| Turbo.ai | Static screenshots + metrics + CTA |

### Current Iframe Problems

**Technical burden** (HeroSection.tsx - 512 lines):
- 15-second timeout before error state
- Complex fullscreen logic (50+ lines)
- Cookie management for demo mode
- Theme sync via postMessage
- Error handling UI

**UX problems**:
- Slow load = bad first impression
- Most visitors don't interact with iframes
- Mobile experience is terrible
- Security sandboxing limits interactivity

**Maintenance burden**:
- Every dashboard change affects demo
- Theme changes require sync logic
- Error states need handling

1. **Shows output value** - visitor sees what they'll get
2. **No maintenance** - static content, easy to update
3. **Fast load** - no iframe, no API calls
4. **Mobile-friendly** - works everywhere
5. **Drives to PLG** - "Upload yours" is natural next step

## Future Considerations

### Mobile App
- Mobile app will be important for acquisition
- Keep homepage flexible for "Download the app" CTA when ready
- Consider app store badges in footer

### UI Evolution
- Don't over-invest in static screenshots that need constant updates
- Prefer abstract visuals or demo data stats over actual UI screenshots
- Design system will evolve; keep landing page decoupled

### Demo Route
- Consider keeping `/demo` as optional deep-dive for engaged visitors
- Not linked prominently, but available for sales/support
