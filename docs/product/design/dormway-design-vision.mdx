# DormWay Design Vision

---

## Core Principle

DormWay is a **high-performance student command center**, not a cheerful productivity carnival.

We design for 2am study sessions, not 2pm marketing presentations. Our users pilot their academic life—they don't visit it.

## ⚠️ Critical Balance: Dark ≠ Monotone

**The Dark Mode Trap:**
Going "dark and premium" can easily become "monotone and hard to scan." This is the #1 mistake when implementing this aesthetic.

**What we DON'T mean by "dark aesthetic":**
- ❌ Everything the same shade of gray
- ❌ No visual hierarchy
- ❌ Colors only for decoration
- ❌ Data visualizations that don't communicate

**What we DO mean:**
- ✅ Dark backgrounds with **strong visual hierarchy** (size, weight, color contrast)
- ✅ Strategic use of **meaningful color** (course colors, status colors, semantic colors)
- ✅ Data visualizations that **show actual information** (not decorative shapes)
- ✅ **Quick scanability** through proper grouping, spacing, and emphasis
- ✅ Visual aids (icons, avatars, charts) that **serve a purpose**

**Example:**
```
BAD: "Study Time - Last 7 days" with unlabeled bars (M T W T F S S)
GOOD: "Study Time - 22.5 hours this week" with labeled bars showing actual hours
      and context like "Goal: 25 hrs/week • 2.5 hrs to go"
```

**Rule of thumb:** If you have to squint or think hard to understand what you're looking at, you've gone too dark.

## The Problem We're Solving Visually

**Bad productivity tools feel like:**
- Elementary school bulletin boards (bright colors, childish)
- Corporate enterprise software (gray, soulless, institutional)
- Social media apps (distracting, dopamine-manipulative)

**DormWay should feel like:**
- A spaceship cockpit (focused, powerful, in control)
- A premium audio interface (tactile, precise, satisfying)
- Arc Browser / Linear / Raycast (modern, Gen Z-native)

## Visual Identity

### Foundation: "Dark Glass + Electric Accents"

**Base Palette:**
```css
/* Depths */
--void-black: #050505;          /* Deepest background */
--surface-black: #0A0A0A;       /* Base canvas */
--card-surface: #1A1A1A;        /* Elevated cards */
--card-surface-hover: #252525;  /* Interactive cards */

/* Borders & Dividers */
--border-subtle: #333333;       /* Card outlines */
--border-interactive: #444444;  /* Hover states */

/* Text Hierarchy */
--text-primary: #EDEDED;        /* Headings, key info */
--text-secondary: #B8B8B8;      /* Body text */
--text-tertiary: #808080;       /* Metadata, labels */
--text-muted: #5A5A5A;          /* Disabled, deemphasized */

/* Brand Accents (The DormWay Signature) */
--electric-violet: #6b21ff;     /* Primary brand */
--hot-pink: #ff5fa1;            /* Secondary brand */
--vivid-orange: #ff7b3f;        /* Tertiary brand */

/* Semantic Colors */
--success: #10b981;             /* Completed tasks */
--warning: #f59e0b;             /* Due soon */
--error: #ef4444;               /* Overdue */
--info: #3b82f6;                /* Informational */
```

### Material Hierarchy

**Layer 1: The Void (Background)**
- Pure black (#050505)
- No gradients, no noise
- Implies infinite depth

**Layer 2: The Surface (Cards)**
- Off-black cards (#1A1A1A)
- Rounded corners (16-24px radius)
- Thin 1px borders (#333333)
- Subtle shadows beneath for depth

**Layer 3: Glassmorphism (Overlays)**
- Backdrop blur (20-40px)
- Semi-transparent backgrounds (rgba(26, 26, 26, 0.8))
- "Shine" effect: Linear gradient on top border
  ```css
  border-top: 1px solid rgba(255, 255, 255, 0.15);
  ```

**Layer 4: Electric Accents (Focus)**
- Brand colors used as **glows, not fills**
- Radial gradients behind interactive elements
- Appears on hover, not by default
- Creates ambient lighting effect

## Interaction Philosophy

### "Physics Over Loops"

**Don't:**
- Constantly moving backgrounds (distraction)
- Looping animations (screensaver vibes)
- Pulsing buttons (desperation)
- Auto-playing anything

**Do:**
- Spring physics on button press
- Hover-triggered glows
- Scroll-triggered reveals
- Smooth 60fps transitions
- Respect reduced motion preferences

### Micro-interactions That Matter

**Button Press:**
```js
scale: 0.98          // Subtle press down
transition: spring   // Bounces back naturally
glow: +20%          // Accent color intensifies
```

**Card Hover:**
```js
border-color: brand  // Subtle color shift
glow: appears       // Radial gradient behind card
elevation: +2px     // Lifts slightly
transition: 200ms   // Fast, responsive
```

**Scroll Reveal:**
```js
opacity: 0 → 1
translateY: 20px → 0
timing: easeOut
stagger: 100ms      // Cascading effect
```

## Typography System

### Font Hierarchy

**Brand Headlines:**
- Font: Custom display font (current: "font-brand")
- Weight: 600-700
- Size: 32-56px (responsive)
- Line height: 1.1
- Letter spacing: -0.02em (tight)
- Color: --text-primary
- Use: Hero headlines, section titles

**Body Text:**
- Font: System font stack (-apple-system, BlinkMacSystemFont)
- Weight: 400 (normal), 500 (emphasis)
- Size: 14-18px
- Line height: 1.6
- Color: --text-secondary
- Use: Paragraphs, descriptions

**UI Labels:**
- Font: System font stack
- Weight: 500-600
- Size: 12-14px
- Letter spacing: 0.01em
- Color: --text-tertiary
- Use: Buttons, chips, metadata

**Code/Monospace:**
- Font: SF Mono, Consolas, monospace
- Weight: 400
- Size: 13-15px
- Color: --electric-violet (for emphasis)
- Use: Course codes, timestamps, technical data

## Layout Patterns

### The Bento Grid

**What it is:**
Compartmentalized layouts where each feature lives in its own rounded card.

**Why it works:**
- Creates visual order
- Feels like a dashboard before login
- Gen Z loves organization + customization
- Scannable at a glance

**Implementation:**
```jsx
<div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
  <Card>Feature 1</Card>
  <Card>Feature 2</Card>
  <Card className="col-span-2">Wide Feature</Card>
</div>
```

### Card Anatomy

**Standard Card:**
```jsx
<div className="
  rounded-3xl              /* Generous corner radius */
  bg-[#1A1A1A]            /* Card surface */
  border border-[#333]     /* Subtle outline */
  p-6                      /* Internal padding */
  shadow-lg                /* Depth */
  backdrop-blur-xl         /* Glass effect */
  hover:border-[#444]      /* Interactive feedback */
  hover:shadow-[0_0_30px_rgba(107,33,255,0.15)]  /* Glow on hover */
  transition-all duration-200
">
  <h3>Card Title</h3>
  <p>Card content</p>
</div>
```

## Color Application Rules

### When to Use Brand Colors

**DO use electric accents for:**
- Primary CTAs
- Active navigation states
- Hover glows behind cards
- Key data points (due dates, scores)
- Status indicators (new, updated)
- Focus states on inputs
- **Course identification** (critical for quick scanning)

**DON'T use electric accents for:**
- Large background areas
- Body text
- Every button
- Borders (keep subtle)
- Illustrations (keep monochrome or minimal)

### Meaningful Color Coding (CRITICAL)

**Course Colors:**
Each course gets a unique color used consistently across the app.

```jsx
// Example: Course color system
const courses = {
  'EECS 281': '#6b21ff',  // Electric violet
  'MATH 215': '#ff5fa1',  // Hot pink
  'PSYCH 111': '#ff7b3f', // Vivid orange
  'ENGLISH 125': '#10b981' // Success green
};

// Use course color for:
- Course code text color
- Left border accent on cards
- Badge backgrounds (10% opacity)
- Calendar event indicators
- Task grouping
```

**Status Colors:**
Colors that communicate state at a glance.

```css
/* Task/Assignment Status */
--status-not-started: #ef4444;  /* Red - urgent */
--status-in-progress: #f59e0b;  /* Orange - active */
--status-completed: #10b981;    /* Green - done */

/* Study Time Goals */
--goal-exceeded: #10b981;       /* Green - 4+ hrs */
--goal-on-track: #f59e0b;       /* Orange - 2-4 hrs */
--goal-behind: #ef4444;         /* Red - < 2 hrs */
```

**Data Visualization:**
Every chart must communicate something actionable.

**Bad:**
```
Study Time: [ambiguous bars with no labels]
```

**Good:**
```
Study Time: 22.5 hours this week
[labeled bars showing Mon 2.5h, Tue 4.0h, ...]
Goal: 25 hrs/week • 2.5 hrs to go
+3.2 hrs vs last week
```

**Key principle:** Color should reduce cognitive load, not add to it.

### Glow Effect Pattern

**Hover Glow Implementation:**
```jsx
<div className="relative group">
  {/* Background glow (hidden by default) */}
  <div className="
    absolute inset-0 -z-10
    bg-gradient-to-r from-[#6b21ff]/20 via-[#ff5fa1]/20 to-[#ff7b3f]/20
    blur-3xl
    opacity-0 group-hover:opacity-100
    transition-opacity duration-300
  "/>

  {/* Card content */}
  <div className="relative bg-[#1A1A1A] rounded-3xl p-6">
    Content here
  </div>
</div>
```

## Animation Guidelines

### Timing Functions

```css
/* Organic, natural feel */
--ease-smooth: cubic-bezier(0.25, 0.1, 0.25, 1);

/* Snappy, responsive */
--ease-snap: cubic-bezier(0.4, 0, 0.2, 1);

/* Spring physics (via Framer Motion) */
--spring-default: { type: "spring", stiffness: 300, damping: 30 };
```

### Duration Standards

- **Micro-interactions:** 150-200ms (button hovers, clicks)
- **Card transitions:** 300-400ms (expand, collapse)
- **Page transitions:** 400-600ms (route changes)
- **Scroll reveals:** 600-800ms (fade in, slide up)

### Respect Accessibility

```jsx
const prefersReducedMotion = useReducedMotion();

<motion.div
  initial={prefersReducedMotion ? false : { opacity: 0, y: 20 }}
  animate={prefersReducedMotion ? undefined : { opacity: 1, y: 0 }}
/>
```

## Responsive Breakpoints

```css
/* Mobile first approach */
--mobile: 0px;       /* Default, no media query */
--tablet: 768px;     /* @media (min-width: 768px) */
--desktop: 1024px;   /* @media (min-width: 1024px) */
--wide: 1440px;      /* @media (min-width: 1440px) */
--ultrawide: 1920px; /* @media (min-width: 1920px) */
```

**Adaptation Strategy:**
- Mobile: Single column, full width cards
- Tablet: 2-column grid, side navigation
- Desktop: 3-column grid, persistent sidebar
- Wide: More whitespace, larger type
- Ultrawide: Max content width, centered layout

## FAQ for Developers

**Q: Can I use light mode?**
A: Yes, support light mode for accessibility, but dark is the default. Users who prefer light will opt-in.

**Q: When should I use the gradient vs. solid brand colors?**
A: Gradients for primary CTAs and hero elements. Solid colors for icons, borders, and small accents.

**Q: How much blur for glassmorphism?**
A: backdrop-blur-xl (20px) for navigation, backdrop-blur-2xl (40px) for modals.

**Q: Can I add more colors?**
A: Semantic colors (success, warning, error) are fine. Don't introduce new brand colors without discussion.

**Q: What about emoji?**
A: Use Lucide icons instead. Emoji feel unprofessional in productivity tools.

**Q: How do I test dark mode?**
A: Set system preference to dark, or use DevTools to simulate. Test with actual users in low-light conditions.

## Design Review Process

**Before shipping visual changes:**

1. **Self-review:** Does this follow the design vision?
2. **Peer review:** Show another developer/designer
3. **Accessibility check:** Test with keyboard, screen reader
4. **Mobile check:** Test on actual device, not just DevTools
5. **User feedback:** If significant change, A/B test with 100+ users

**Red flags to reject:**
- Bright, saturated backgrounds
- Looping animations without purpose
- Comic Sans (obviously)
- Gradient borders (hard to maintain)
- Clashing with brand colors
- Not keyboard accessible

## Evolution, Not Revolution

This document describes our **target state**, not a "throw away everything" mandate.

**Current state (Nov 2025):**
- Light mode default with dark mode support
- Aurora gradient backgrounds
- Vibrant colors throughout
- Good foundation in place

**Target state (Q1 2026):**
- Dark mode default with light mode support
- Ambient glows behind interactive elements
- Brand colors as accents, not backgrounds
- Refined, premium feel

**How we get there:**
- Incremental updates, not big bang redesign
- A/B test major changes
- Preserve brand recognition
- Listen to user feedback

## Conclusion

**DormWay is a high-performance student command center.**

We design with the confidence that our users are capable, intelligent, and deserve tools that respect their time. We use dark backgrounds and ambient glows because students work at night. We use physics-based animations because digital interfaces should feel tactile. We use our brand colors sparingly because accent lights are more premium than neon signs.

**Every design decision should ask:**
*"Does this help a student at 2am see what's due tomorrow?"*

If the answer is no, cut it.
