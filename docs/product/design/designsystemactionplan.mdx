# Design System Unification - Action Plan

## Executive Summary

Two well-maintained but diverging design systems need coordination:
- **iOS**: Complete, generated, accessible-first, limited themes
- **Web**: Flexible, hand-maintained, theme-rich, incomplete typography

This plan outlines how to unify them while preserving platform strengths.

### 3. Naming Conventions Document (1.5 hours)

**Create File**: `DESIGN_TOKENS_NAMING.md`

```markdown
# Design System Token Naming Conventions

## iOS Naming (DormWayDesignSystem)

Namespace structure:
- `DS.Color.Palette.Brand` - Raw colors
- `DS.Color.Semantic.Text` - Role-based colors
- `DS.Spacing` - All spacing values
- `DS.Radius` - Corner radius values

Naming pattern: contextual, semantic
- `primary`, `primaryLg`, `primaryMd` (color variants)
- `eveningPrimary` (contextual color)
- `cardInternal` (component-specific)

## Web Naming (CSS Variables)

Namespace structure:
- `--ds-` prefix for all tokens
- Hyphen-separated: `--ds-surface-1`
- Numeric suffixes for scales: `-1`, `-2`, `-3`

Naming pattern: role-based, hierarchical
- `--ds-surface-0` through `--ds-surface-3`
- `--ds-text-primary`, `--ds-text-secondary`, etc.
- `--ds-shadow-sm` through `--ds-shadow-xl`

## Unified Naming Rules

### Colors

Format: `{context}-{intent}-{variant}`

Examples:
- `primary` (primary brand)
- `secondary` (secondary brand)
- `success`, `warning`, `error`, `info` (status)
- `text-primary`, `text-secondary` (semantic)
- `surface-0`, `surface-1` (depth)

### Spacing

Format: `{size}`

Scale: `xs` (4px) ‚Üí `sm` (8px) ‚Üí `md` (16px) ‚Üí `lg` (24px) ‚Üí `xl` (32px) ‚Üí `xxl` (48px) ‚Üí `xxxl` (64px)

Component-specific: `{component}-{part}`
- `card-internal`: 20px (internal card padding)
- `section-gap`: 28px (between sections)

### Typography

Format: `{type}-{size}`

Types: `h1`, `h2`, `h3`, `body`, `bodySmall`, `caption`, `label`, `button`

Or: `{role}` - `heading`, `body`, `display`, `mono`

### Radius

Format: `{size}`

Scale: `none` (0) ‚Üí `sm` (8px) ‚Üí `md` (12px) ‚Üí `lg` (16px) ‚Üí `xl` (24px) ‚Üí `full` (9999px)

Component-specific: `{component}-radius`
- `card-radius`: 16px
- `chip-radius`: 9999px

## Migration Path

For Web, rename existing tokens to match iOS:
- `spacing-2xl` ‚Üí `spacing-xxl`
- `spacing-3xl` ‚Üí `spacing-xxxl`
- `radius-lg`: 16px (currently 1rem) - needs adjustment

For iOS, consider adding theme context names:
- Add `glass-primary`, `paper-primary` if glass/paper themes added

## Documentation

- Add this guide to project docs
- Create token generator script to validate naming
- Add linter rule to prevent new tokens without naming review
```
