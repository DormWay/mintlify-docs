## Design Philosophy

**Core Principle**: Students should feel ownership over their day through direct manipulation, with immediate proof of system reliability.

### Design Values
- **Tactile Planning** - Physical drag/drop creates stronger commitment than abstract lists
- **Transparent Intelligence** - Every claim shows its reasoning and confidence
- **Contextual Depth** - Expand details in place rather than navigation
- **Respectful Defaults** - System suggests but user decides

## Interaction Patterns

### Timeline Interactions

**Desktop Drag & Drop**
1. User hovers over task in bank â†’ drag cursor appears
2. User drags task â†’ timeline shows drop zones (green) and conflicts (amber)
3. User drops task â†’ work block appears with default duration
4. Auto-snap to 15-minute increments, avoid class overlaps

**Mobile Alternative**
1. User taps task in bank â†’ "Add to Timeline" button appears
2. User taps "Add to Timeline" â†’ time picker modal opens
3. User selects start time and duration â†’ work block creates
4. Conflict warnings shown before confirmation

**Block Editing (All Platforms)**
- Tap block â†’ inline edit card appears below/adjacent
- Resize handles on desktop (drag edges)
- Mobile: duration buttons (+15min, +30min, -15min)

## Visual Design System

### Color & Status

**Work Block States**
- Planned: `bg-blue-50 border-blue-200 text-blue-900`
- In Progress: `bg-amber-50 border-amber-200 text-amber-900`
- Completed: `bg-green-50 border-green-200 text-green-700`
- Overdue: `bg-red-50 border-red-200 text-red-900`

**Timeline Elements**
- Hour lines: `border-gray-200`
- Now line: `border-red-400 bg-red-50` with subtle animation
- Drop zones: `border-green-400 bg-green-50` (dashed border)
- Conflicts: `border-amber-400 bg-amber-50`

**Trust Indicators**
- High confidence (80-100%): Green accent
- Medium confidence (60-79%): Amber accent
- Low confidence (below 60%): Red accent

### Typography

**Timeline Text**
- Event titles: `text-sm font-medium text-gray-900`
- Event details: `text-xs text-gray-600`
- Time labels: `text-xs font-mono text-gray-500`

**Card Text**
- Card titles: `text-lg font-semibold text-gray-900`
- Body text: `text-sm text-gray-700`
- Meta text: `text-xs text-gray-500`

### Spacing & Layout

**Timeline Grid**
- Hour height: 60px
- Event padding: 8px
- Block margin: 2px vertical
- Timeline gutter: 60px for time labels

**Card Spacing**
- Internal padding: 16px
- Element spacing: 12px vertical
- Button spacing: 8px horizontal

## Animation & Micro-interactions

### Timeline Animations
- Block creation: Scale in from 0.8 to 1.0 (200ms ease-out)
- Drag feedback: Subtle shadow and scale (1.02x)
- Drop success: Brief green flash on drop zone
- Collision warning: Gentle shake animation

### Card Transitions
- Open: Slide up (mobile) or fade in (desktop) - 150ms
- Close: Slide down (mobile) or fade out (desktop) - 100ms
- Action buttons: Subtle scale on hover (1.05x)

### Trust Surface Feedback
- Source chip tap: Brief scale (0.95x) for tactile feedback
- Re-scan: Spinner on button, timestamp updates
- Confidence change: Color transition over 300ms

## Accessibility Features

### Screen Reader Support
- Timeline: Proper heading structure and landmark regions
- Work blocks: Clear labels with time, duration, and status
- Trust panels: Detailed descriptions of confidence and sources
- Keyboard navigation: Full functionality without mouse

### Keyboard Shortcuts
```
Global (when timeline focused):
- n: New work block
- t: Focus task bank
- r: Open "Really?" for current view
- /: Focus search/filter

Timeline specific:
- Arrow keys: Navigate between blocks
- Enter: Edit selected block
- Delete: Remove selected block
- Space: Toggle block completion

Task bank:
- Enter: Add task to timeline
- Tab: Navigate between sections
```

### Color & Contrast
- All text meets WCAG AA standards (4.5:1 minimum)
- Color never sole indicator of state
- High contrast mode support
- Reduced motion preferences respected

## Performance Guidelines

### Rendering Targets
- Timeline initial render: `<200ms`
- Block creation: `<100ms`
- Card open: `<150ms`
- Drag operation: 60fps smoothness

### Loading States
- Skeleton screens for initial data load
- Progressive enhancement for secondary data
- Optimistic updates for user actions
- Clear progress indicators for long operations

## Usability Testing Focus Areas

1. **Drag & Drop Discoverability**
   - Can users understand the interaction without instruction?
   - How do users recover from drag mistakes?

2. **Trust Surface Engagement**
   - Do users click "Really?" buttons?
   - Does seeing sources increase confidence?

3. **Mobile Timeline Navigation**
   - Is the alternative to drag/drop intuitive?
   - Can users efficiently manage multiple blocks?

4. **Cognitive Load**
   - Does the interface feel overwhelming?
   - Can users maintain focus on planning vs. managing the tool?
